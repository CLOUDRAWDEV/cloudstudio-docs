## Resource Overview

Azure Database for PostgreSQL is a fully managed relational database service provided by Microsoft Azure, based on the widely used open-source PostgreSQL database engine. 
It allows you to easily deploy, manage, and scale PostgreSQL databases in the cloud without worrying about the underlying infrastructure. 
With Azure Database for PostgreSQL, Microsoft handles tasks such as patching, backups, and monitoring, enabling you to create and manage PostgreSQL databases with minimal management overhead. 
Additionally, you can take advantage of features like high availability, automatic backups, and point-in-time recovery.

## Resource Setting Values
Here is the revised English version:

-`version`: The version of PostgreSQL to be used, e.g., `9.5`, `9.6`, `10`, `10.0`, `10.2`, `11`.

-`sku_name`: The pricing tier name for PostgreSQL, a combination of Tier + Compute Generation + Number of Cores, e.g., `azureDatabaseSku`.

-`storage_mb`: The maximum capacity (in MB) for the PostgreSQL server, ranging from `5120` to `16777216`.

-`create_mode`: The storage IOPS mode for MySQL Flexible Server, e.g., `Default`, `Replica`, `GeoRestore`, `PointInTimeRestore`.

-`admin_username`: The administrator username for the server when the creation mode is `default`.

-`admin_password`: The administrator login password for the server when the creation mode is `default`.

-`public_network_access_enabled`: Whether public network access is allowed for the server, `true` or `false`.

-`ssl_enforcement_enabled`: Whether encrypted connections (SSL) are enforced, `true` or `false`.

-`minimal_tls_version`: The minimum TLS version supported by the server, e.g., `TLS1_2`, `TLS1_1`, `TLS1_0`, `TLSEnforcementDisabled`.

-`database`
    -`charset`: The character set for the PostgreSQL database, e.g., `azurePostgresqlCharset`.
    -`collation`: The collation for the PostgreSQL database, e.g., `azurePostgresqlCollation`.

-`firewall_rule`
    -`start_ip_address`: The starting IP address for the firewall rule.
    -`end_ip_address`: The ending IP address for the firewall rule.

-`tag`: A tag used to categorize resources, up to 512 characters, in the format `key:value`.
