## Resource Overview

Here's the revised text in English:

---

Azure Database for MySQL is a fully managed relational database service offered by Microsoft Azure, based on the widely used open-source MySQL database engine. 
You can easily deploy, manage, and scale MySQL databases in the cloud without worrying about the underlying infrastructure. 
With Azure Database for MySQL, Microsoft handles tasks such as patching, backups, and monitoring, allowing you to create and manage MySQL databases with minimal management overhead.
Additionally, you can take advantage of features like high availability, automated backups, and point-in-time restore.

## Resource Configuration Values

-   `version` : The MySQL version to use - `5.7`, `5.6`, `8.0`

-   `sku_name` : The pricing tier name for MySQL, a combination of Tier + Compute Generation + Number of Cores - `azureDatabaseSku`

-   `storage_mb` : The capacity of the MySQL server in MB - `5120` ~ `16777216`

-   `admin_username` : The administrator username for the server when the creation mode is default

-   `admin_password` : The administrator login password for the server when the creation mode is default

-   `public_network_access_enabled` : Whether to allow public network access to the server - `true` , `false`

-   `ssl_enforcement_enabled` : Whether encrypted connections (SSL) are applied - `true` , `false`

-   `minimal_tls_version` : The minimum TLS version supported by the server - `TLS1_2` , `TLS1_1` , `TLS1_0` , `TLSEnforcementDisabled`

-   `threat_detection_enabled` : Whether to detect anomalous activities that indicate abnormal and potentially harmful attempts - `true` , `false`

-   `database`

    -   `charset` : The character set for the MySQL database - `azureMysqlChartSet`
    -   `collation` : The collation method for data in the Microsoft SQL database - `azureMysqlCol`

-   `firewall_rule`
    -   `start_ip_address` : The starting IP address for the firewall rule
    -   `end_ip_address` : The ending IP address for the firewall rule
-   `tag` : A tag used to categorize resources - up to 512 characters `key`:`value`
