## 리소스 개요

Azure Monitor activity log alert는 Azure 활동 로그의 특정 이벤트 또는 변경 사항을 기반으로 경고를 생성할 수 있는 Azure의 알림 서비스 유형입니다.
Azure activity log laert는 Azure 구독의 리소스에서 수행된 모든 작업에 대한 포괄적인 로그를 제공합니다.
리소스가 생성, 수정 또는 삭제될 때와 같은 특정 조건이 충족될 때 알림을 보내도록 활동 로그 경고를 구성할 수 있습니다.
특정 오류 메시지를 기반으로 하거나 특정 임계값을 초과하는 경우(예: 실패한 요청 수가 많거나 트래픽이 갑자기 증가하는 경우)에 따라 경고를 생성할 수도 있습니다.

## 리소스 설정 값

-   `criteria_category` : 필터링할 작업 범주 - `Administrative`,`Autoscale`,`Policy`,`Recommendation`,`Security`,`ResourceHealth`,`ServiceHealth`

-   `criteria_level` : 활동 로그 경고가 모니터링할 이벤트의 심각도 수준 - `Verbose`,`Informational`,`Warning`,`Error`,`Critical`

-   `target_resource_name` : 활동 로그 경고의 모니터링을 받는 리소스 이름
-   `criteria_status` : 활동 로그 경고가 모니터링할 이벤트의 상태 값 - `Started`,`Failed`,`Succeeded`

-   `linked_action_group_name` : 경고 규칙을 적용할 작업 그룹 이름
-   `tag` : 리소스를 카테고리로 분류하기 위한 태그 - 최대 512자, `key`:`value`
