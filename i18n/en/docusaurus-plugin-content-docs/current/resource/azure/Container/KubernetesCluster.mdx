## Related Resources

-   `dns_prefix` : DNS prefix to be used with the FQDN of the hosted Kubernetes API server.

-   `sku_tier` : Pricing tier for using the Kubernetes cluster. - `Free`, `Paid`

-   `enable_public_network_access` : - `true`, `false`

-   `api_server_authorized_ip_ranges` : Authorized IP address ranges to secure access to the API server.
-   `default_node_pool`

    -   `node_pool_name` : Name of the Node Pool to be created in the Kubernetes cluster
    -   `enable_auto_scaling` : Enable automatic scaling for the cluster to ensure it runs with the appropriate number of nodes
    -   `node_count` : Number of nodes in the Node Pool - `1`~ `1000`
    -   `vm_size` : Size of the virtual machines that will form the nodes in the Node Pool
    -   `os_disk_size_gb` : Size of the OS disk for the virtual machines (in GB) - `0`~ `114000`
    -   `max_pods` : Maximum number of Pods that can run on a node - `10`~ `250`
    -   `enable_node_public_ip` : Enable or disable the use of public IP addresses per node - `true`, `false`

-   `authentication_authorization` : Enable role-based access control to restrict access to cluster resources - `local_accounts`, `azure_rbac`, `kubernetes_rbac`

-   `network_policy` : Network policy to restrict access to specific Pods - `calico`, `azure`

-   `network_plugin` : Configure the virtual network using either kubenet or Azure CNI - `kubenet`, `azure`

-   `included_subnet_name` : Name of the subnet where the Kubernetes Cluster will be deployed

-   `load_balancer_sku` : Pricing tier for the Load Balancer - `standard`, `basic`

-   `pod_cidr` : CIDR for using Pod IP addresses

-   `outbound_type` : Outbound routing type - `loadBalancer`, `userDefinedRouting`

-   `cluster_node_pool`

    -   `node_mode` : System or user mode for the Node Pool - `User`, `System`
    -   `node_os_type` : OS type for the VMs in the Node Pool - `Linux`, `Windows`
    -   `enable_auto_scaling` : Enable automatic scaling to ensure the cluster runs with the appropriate number of nodes - `true`, `false`
    -   `node_count` : Number of nodes in the Node Pool (for user mode) - `1`~ `1000`
    -   `max_node_count` : Maximum number of nodes in the Node Pool - `0`~ `1000`
    -   `min_node_count` : Minimum number of nodes in the Node Pool - `0`~ `1000`
    -   `vm_size` : Size of the virtual machines that form the nodes in the Node Pool
    -   `os_disk_size_gb` : Size of the OS disk for the virtual machines (in GB) - `4`~ `11400`
    -   `max_pods` : Maximum number of Pods that can run on a node.
    -   `enable_node_public_ip` : Enable or disable the use of public IP addresses per node - `true`, `false`
    -   `public_ip_prefix_name`

-   `tag` : Tag for categorizing resources - up to  512 characters, `key`:`value`
