## Resource Overview

Azure Cosmos DB Gremlin is a fully managed graph database service provided by Microsoft Azure. 
Gremlin is a graph traversal language that allows users to explore and query graph data in Azure Cosmos DB. 
It supports a wide range of graph operations, including vertex and edge creation, adding and removing properties, and performing complex graph traversals.

## Resource Setting Values

-   `capabilities_list` : List of Cosmos DB APIs to be used - `EnableServerless`, `EnableAggregationPipeline`

-   `enable_multiple_write_locations` : Enable multi-region writes to leverage provisioned throughput for databases and containers worldwide - `true`, `false`

-   `public_network_access_enabled` : Enable public access to the account via public IP addresses or service endpoints - `true`, `false`

-   `enable_automatic_failover` : Enable automatic failover - `true`, `false`

-   `virtual_network_filter_enabled` : Enable filtering to allow access to the account only from specific subnets within a virtual network - `true`, `false`

-   `network_acl_bypass_for_azure_services` : Enable ACL bypass for Azure services - `true`, `false`

-   `geo_location`

    -   `failover_priority` : Priority for failover in the region - `0`~ `0`
    -   `zone_redundant` : Allow automatic zone replication for fault tolerance - `true`~ `false`

-   `consistency_policy`

    -   `consistency_level` : Consistency level for Cassandra DB to configure database availability and performance - `Eventual`, `Session`, `Strong`, `ConsistentPrefix`, `BoundedStaleness`
    -   `max_interval_in_seconds` : Allowed staleness time (in seconds) when the consistency level is Bounded Staleness - `5`~ `86400`
    -   `max_staleness_prefix` : Maximum number of stale requests allowed when the consistency level is Bounded Staleness - `10`~`2147483647`

-   `backup`

    -   `type` : Backup policy type - `Continuous`, `Periodic`
    -   `interval_in_minutes` : Frequency of backups in minutes, within the range of 60 ~ 1440 - `60`~ `1440`
    -   `retention_in_hours` : Duration for which backups are retained (in hours), within the range of 8 ~ 720 - `8`~`720`

-   `gremlin_database`

    -   `throughput` : Throughput for the Gremlin database - `400`~ `1000000`
    -   `enable_autoscale` : Whether to enable auto-scaling for the Gremlin database - `true`, `false`

-   `gremlin_graph`

    -   `partition_key_version` : Partition key version, either 1 or 2 - `1`~ `2`
    -   `partition_key_path` : Path for the partition key.
    -   `throughput` : Throughput for the Gremlin graph - `400`~ `100000`
    -   `automatic` : Enable automatic indexing - `true`, `false`
    -   `indexing_mode` : Indexing type setting - `consistent` , `lazy`, `none`
    -   `included_paths` : List of paths included in indexing
    -   `excluded_paths` : List of paths excluded from indexing
    -   `conflict_resoulution_mode` : Conflict resolution type - `LastWriterWins` , `Custom`
    -   `unique_key_paths` : List of unique key paths for data integrity
    -   `enable_autoscale` : Processing of Gremlin graph - `true`, `false`
    -   `included_gremlin_database_name` : Name of the Gremlin database where the Gremlin graph will be created

-   `tag` : Tag for categorizing resources - up to 512 characters, `key`:`value`
