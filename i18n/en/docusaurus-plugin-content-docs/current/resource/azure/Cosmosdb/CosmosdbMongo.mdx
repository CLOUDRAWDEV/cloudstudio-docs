## Resource Overview

Cosmos DB is a multi-model distributed database provided by Microsoft Azure that supports NoSQL document, key-value, graph, and column-family data models. 
It is a globally distributed database that replicates data across multiple regions to provide low latency and high availability.

### Parent Resource

-   [**Resource Group**](https://docs.cstudio.app/docs/resource/azure/Base/ResourceGroup)

## Resource Configuration Values

-   `capabilities_list` : List of Cosmos DB APIs to be used - `EnableServerless`, `EnableAggregationPipeline`

-   `enable_multiple_write_locations` : Enable multi-region writes to utilize provisioned throughput for databases and containers worldwide - `true`, `false`

-   `public_network_access_enabled` : Enable public access to the account via public IP addresses or service endpoints - `true`, `false`

-   `enable_automatic_failover` : Enable automatic failover - `true`, `false`

-   `virtual_network_filter_enabled` : Enable filtering to allow access to the account only from specific subnets within a virtual network - `true`, `false`

-   `network_acl_bypass_for_azure_services` : Enable ACL bypass for Azure services - `true`, `false`

-   `geo_location`

    -   `failover_priority` : Priority for failover in the region - `0`~ `0`
    -   `zone_redundant` : Allow automatic zone replication for fault tolerance - `true`~ `false`

-   `consistency_policy`

    -   `consistency_level` : Consistency level for Cassandra DB to configure database availability and performance - `Eventual`, `Session`, `Strong`, `ConsistentPrefix`, `BoundedStaleness`
    -   `max_interval_in_seconds` : Allowed staleness time (in seconds) when the consistency level is Bounded Staleness - `5`~ `86400`
    -   `max_staleness_prefix` : Maximum number of stale requests allowed when the consistency level is Bounded Staleness - `10`~`2147483647`

-   `backup`

    -   `type` : Backup policy type - `Continuous`, `Periodic`
    -   `interval_in_minutes` : Frequency of backups in minutes, within the range of 60 ~ 1440 - `60`~ `1440`
    -   `retention_in_hours` : Duration for which backups are retained (in hours), within the range of 8 ~ 720 - `8`~`720`

-   `mongo_database`

    -   `throughput` : Throughput for the MongoDB database - `400`~ `1000000`
    -   `enable_autoscale` : Whether to enable auto-scaling for the MongoDB database - `true`, `false`

-   `mongo_collection`

    -   `ttl_seconds` : Default Time To Live (TTL) for the MongoDB collection (in seconds)
    -   `shard_key` : Name of the shard key for MongoDB collection sharding.
    -   `throughput` : Throughput for the MongoDB collection - `400`~ `100000`
    -   `enable_autoscale` : Whether to enable auto-scaling for the MongoDB collection - `true`, `false`
    -   `included_mongo_database_name` : Name of the MongoDB database where the collection will be created

-   `tag` : Tag for categorizing resources - up to 512 characters, `key`:`value`
