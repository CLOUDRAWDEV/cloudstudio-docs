## Resource Overview

Azure Sentinel is a cloud-native SIEM (Security Information and Event Management) service offered by Microsoft Azure. 
It provides comprehensive security solutions by delivering intelligent security analytics and threat intelligence across the enterprise to protect IT infrastructure and assets from cyber threats. 
Azure Sentinel collects data from various sources, including servers, networks, applications, and devices, to offer an integrated view of security threats across the organization. 
It uses advanced analytics and machine learning algorithms to detect and respond to threats in real time.

### Connected Resources

Here is the corrected text:

-   `linked_workspace_name`: The name of the Log Analytics Workspace to which Sentinel will be connected.

-   `connect_security_center`: Whether to connect the Microsoft Defender for Cloud Data Connector to Sentinel - `true`, `false`

-   `connect_threat_intelligence`: Whether to connect the Threat Intelligence Platform (preview) Data Connector to Sentinel - `true`, `false`

-   `connect_advanced_threat_protection`: Whether to connect the Microsoft Defender for Identity Data Connector to Sentinel - `true`, `false`

-   `connect_microsoft_defender`: Whether to connect the Microsoft Defender Data Connector for endpoints to Sentinel - `true`, `false`

-   `connect_office_365`: Whether to connect the Office 365 Data Connector to Sentinel - `true`, `false`

-   `connect_cloud_app_security`: Whether to connect the Cloud App Security Data Connector to Sentinel - `true`, `false`

-   `fusion_rule_rule_guid`: The GUID for the Microsoft Defender for Cloud Apps Data Connector to be connected to Sentinel.

-   `behavior_analytics_rule_guid`: A list of GUIDs for Sentinel ML Behavior Analytics Rules.

-   `alert_rule_incident`:

    -   `display_name`: The display name of the MS Security Incident Alert Rule in Sentinel.
    -   `product`: The Microsoft security service that generates the alert.
    -   `severity`: The severity level of the MS Security Incident Alert - `High`, `Medium`, `Low`, `Informational`.

-   `alert_rule_scheduled`:

    -   `display_name`: The display name of the Scheduled Query Rule in Sentinel.
    -   `severity`: The severity level of the Scheduled Alert Rule in Sentinel - `High`, `Medium`, `Low`, `Informational`.
    -   `query`: The query statement for the Scheduled Alert Rule in Sentinel.

-   `tag`: A tag for categorizing the resource - up to 512 characters, `key:value`.
