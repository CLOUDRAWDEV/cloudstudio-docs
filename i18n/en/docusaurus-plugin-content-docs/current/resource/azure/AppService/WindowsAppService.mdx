## Resource Overview

Azure App Service is a PaaS offering that allows you to build and host web apps, mobile backends, and RESTful APIs using your chosen programming language, without needing to manage the infrastructure. 
App Service provides auto-scaling and high availability and supports both Windows and Linux operating systems. 
It can be automatically deployed from GitHub, Azure DevOps, or Git repositories.

## Resource Configuration Values

-   `linked_service_plan_name`: The plan policy applied to the App Service
-   `delegated_subnet_name`: The subnet name for virtual network delegation
-   `client_cert_enabled`: Enable client certificates - `true`, `false`
-   `auth_settings_enabled`: Enable authentication to ensure all communications from the app are authenticated - `true`, `false`
-   `detailed_error_messages_enabled`: Include detailed error messages on error pages when an HTTP error occurs - `true`, `false`
-   `failed_request_tracing_enabled`: Enable detailed tracing information logging for failed requests - `true`, `false`
-   `retention_in_days`: Number of days to retain logs related to HTTP requests - `0` to `99999`
-   `retention_in_mb`: Storage capacity (in MB) to retain logs related to HTTP requests - `25` to `100`
-   `ftps_state`: Apply TLS/SSL to FTP for security - `Disabled`, `FtpsOnly`, `AllAllowed`
-   `http2_enabled`: Use the latest version of the HTTP protocol for security - `true`, `false`
-   `tag`: Tag for categorizing resources - up to 512 characters, `key:value`
