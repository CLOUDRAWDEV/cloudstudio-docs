## Resource Overview

An availability set is a logical grouping of VMs in Azure that allows you to understand how applications are built to provide redundancy and availability. 
To deliver high availability and meet the 99.95% Azure SLA, it is recommended to create more than one VM within an availability set. 
There is no cost associated with the availability set itself; you only pay for each VM instance that you create.

## Resource Configuration Values

- `update_domain_count`: The number of update domains used to reboot virtual machines in a group simultaneously, ranging from `1` ~ `20`.
- `fault_domain_count`: The number of fault domains, ranging from `1` ~ `3`.
- `tag`: A tag used to categorize resources, up to 512 characters, in the format `key:value`.
