## Resource Overview

An Azure Linux Virtual Machine Scale Set is a group of identical virtual machines running the Linux operating system that can automatically scale in or out based on demand. 
You can manually adjust the number of VMs in the scale set or define rules to automatically resize based on resource usage (such as CPU, memory requirements, or network traffic). 
Azure load balancers then distribute traffic across the VM instances in the scale set.

## Resource Configuration Values

- `included_vnet_name`: The name of the Virtual Network that includes the subnet.
- `orchestration_mode`: The orchestration mode that determines how the Scale Set manages VMs.
- `platform_fault_domain_count`: The number of groups of VMs sharing common power and network switches.
- `vmss_sku`: The pricing tier of the Virtual Machine Scale Set.
- `instances`: The number of virtual machines in the Scale Set, ranging from `1` ~ `1000`.
- `source_image`:
  - `enabled_custom_image`: Whether to use a custom image ID, `true` or `false`.
  - `image_id`: The ID of the image used for VM creation.
  - `publisher`: The publisher of the image used for VM creation.
  - `offer`: The offer ID of the image used for VM creation.
  - `sku`: The pricing tier of the image used for VM creation.
- `enable_ssh_key_authentication`: Whether to use an SSH key for administrator account authentication.
- `admin_username`: The administrator username for the VM.
- `admin_password`: The administrator password for the VM.
- `os_disk_caching`: The type of caching for the internal OS disk, options are `ReadWrite`, `ReadOnly`, `None`.
- `storage_account_type`: The type of storage used for backing up the internal OS disk, options are `Standard_LRS`, `StandardSSD_LRS`, `Premium_LRS`.
- `disk_size_gb`: The size of the internal OS disk in GB, ranging from `1` ~ `11400`.
- `network_interface`: The name of the Network Interface connected to the Virtual Machine.
- `tag`: A tag used to categorize resources, up to 512 characters, in the format `key:value`.
