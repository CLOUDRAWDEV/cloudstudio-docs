## Resource Overview

You can create a file system, mount it to an EC2 instance, read and write data to the file system from the EC2 instance, and share files.

## Resource Configuration Values

-   `create_token`: A unique name used as a reference when creating an Elastic File System to ensure the file system's creation.
-   `availability_zone_name`: The AWS Availability Zone where the file system will be created.
-   `enabled_encrypted`: Indicates whether the file system is encrypted - `false`, `true`.
-   `throughput_mode`: The throughput mode for the file system - `provisioned`, `elastic`, `bursting`.
-   `provisioned_throughput_in_mibps`: The throughput (in MiB/s) to provision for the file system - `1-3414`.
-   `performance_mode`: The performance mode for the file system - `generalPurpose`, `maxIO`.
-   `lifecycle_policy`: Lifecycle management that automatically reduces costs as access patterns change.
-   `lifecycle_policy.transition_to_ia`: The time it takes to transition files to the IA (Infrequent Access) storage class - `1`, `7`, `14`, `30`, `60`.
-   `lifecycle_policy.transition_to_primary_storage_class`: The policy used to transition files from the access storage class back to the primary storage class - `None`, `FirstAccess`.
-   `mount_target_subnets`: A list of subnets to which mount targets will be added.
-   `file_system_policy`: A JSON-formatted file system policy for the EFS file system.
-   `enable_backup_policy`: Indicates whether the backup policy is enabled - `false`, `true`.
-   `enable_replica`: Indicates whether to create a replica - `false`, `true`.
-   `access_point`: An application-specific entry point in the EFS file system that simplifies managing application access to shared datasets.
-   `access_point.root_directory_path`: The path in the EFS file system exposed as the root directory to NFS clients accessing the file system using this access point.
-   `access_point.posix_user_uid`: The POSIX user ID used for all file system operations performed via this access point - `below 4294967295`.
-   `access_point.posix_user_gid`: The POSIX group ID used for all file system operations performed via this access point - `below 4294967295`.
-   `access_point.posix_user_secondary_gids`: The secondary POSIX group IDs used for all file system operations via this access point.
-   `access_point.root_owner_uid`: The POSIX user ID applied to the root directory - `below 4294967295`.
-   `access_point.root_owner_gid`: The POSIX group ID applied to the root directory - `below 4294967295`.
-   `access_point.root_permissions`: The POSIX permissions, in octal format, applied to the root directory.
