## Resource Overview

Amazon DynamoDB is a fully managed NoSQL database service that provides fast and predictable performance with seamless scalability. 
With Amazon DynamoDB, you can create database tables to store and retrieve any amount of data, handling any level of request traffic. 
It automatically distributes data and traffic across a sufficient number of servers to ensure consistent and fast performance while accommodating the request capacity and data volume specified by the customer.

## Resource Configuration Values

-   `billing_mode`: The pricing method and capacity management for read and write throughput in DynamoDB - `PROVISIONED`, `PAY_PER_REQUEST`
-   `read_capacity`: The number of read units for the DynamoDB table
-   `write_capacity`: The number of write units for the DynamoDB table
-   `hash_key`: The attribute to be used as the hash (partition) key
-   `hash_key_type`: The data type of the hash (partition) key attribute - `S` (String), `N` (Number), `B` (Binary)
-   `range_key`: The attribute to be used as the range (sort) key
-   `range_key_type`: The data type of the range (sort) key attribute - `S` (String), `N` (Number), `B` (Binary)
-   `point_in_time_recovery_enabled`: Whether point-in-time recovery is enabled - `true`, `false`
-   `server_side_encryption`: The server-side encryption settings
  -     `encryption_enabled`: Whether to enable encryption at rest using AWS-managed KMS customer master keys - `true`, `false`
