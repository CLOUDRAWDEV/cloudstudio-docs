## Resource Overview

Amazon CloudFront accelerates the distribution of static and dynamic web content such as .html, .css, .php, images, and media files. 
When a user requests content, CloudFront serves it through a network of edge locations around the world, providing low latency and high performance.

## Resource Configuration Values

-   `enabled` : Whether the distribution is enabled to accept end-user requests for content - `true`, `false`
-   `origin` : Identifier of the origin
-   `cache_policy` : Cache Policy name
    -   `min_ttl` : Minimum amount of time (in seconds) that an object should remain in the CloudFront cache
    -   `max_ttl` : Maximum amount of time (in seconds) that an object should remain in the CloudFront cache
    -   `default_ttl` : Default amount of time (in seconds) that an object should remain in the CloudFront cache
    -   `cookie_behavior` : Determines whether cookies from end-user requests are included in the cache key and automatically included in requests CloudFront sends to the origin - `whitelist`, `all`, `none`, `allExcept`
    -   `cookies_items` : List of cookie items names
    -   `header_behavior` : Determines whether HTTP headers are included in the cache key and automatically included in requests CloudFront sends to the origin - `whitelist`, `none`
    -   `headers_items` : List of header items names
    -   `query_string_behavior` : Determines whether URL query strings from end-user requests are included in the cache key and automatically included in requests CloudFront sends to the origin - `whitelist`, `all`, `none`, `allExcept`
    -   `query_string_items` : List of query items names
-   `cache_behavior`
    -   `path_pattern` : Pattern specifying the requests to which the cache behavior applies
    -   `compress` : Whether CloudFront should automatically compress content for web requests - `true`, `false`
    -   `viewer_protocol_policy` : Specifies the protocol that can be used to access the files from the origin specified in TargetOriginId when the request matches the path pattern - `allow-all`, `https-only`, `redirect-to-https`
    -   `allowed_methods` : Controls the HTTP methods that CloudFront processes and forwards to an Amazon S3 bucket or custom origin - `GET`, `HEAD`, `OPTIONS`, `PATCH`, `POST`, `PUT`, `DELETE`
    -   `cached_methods` : Controls whether CloudFront caches responses for requests using the specified HTTP methods - `GET`, `HEAD`, `OPTIONS`, `PATCH`, `POST`, `PUT`, `DELETE`
    -   `origin_name` : Origin name to which CloudFront routes requests when the path pattern matches
    -   `cache_policy_name` : Cache Policy ID to associate with the Cache Behavior

-   `restriction`
    -   `type` : Method used to restrict content distribution by country - `whitelist`, `blacklist`, `none`
    -   `locations` : ISO 3166-1-alpha-2 codes of locations where CloudFront should or should not distribute content

-   `viewer_certificate`
    -   `default_certificate_enabled` : Whether to use HTTPS for requests from end-users and use the CloudFront domain name for the distribution - `true`, `false`

-   `price_class` : Price class of the CloudFront distribution - `PriceClass_100`, `PriceClass_200`, `PriceClass_All`
-   `http_version` : Maximum HTTP version supported by the CloudFront distribution - `http1.1`, `http2`
-   `is_ipv6_enabled` : Whether IPv6 is enabled for the CloudFront distribution - `true`, `false`

-   `logging`
    -   `logging_enabled` : Whether logging is enabled - `true`, `false`

-   `linked_waf_web_acl_name` : Name of the WAF WEB ACL to use if security protection is enabled
