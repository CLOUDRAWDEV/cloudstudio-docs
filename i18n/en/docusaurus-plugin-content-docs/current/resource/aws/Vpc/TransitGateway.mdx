## Resource Overview

The Transit Gateway is a network transit hub that can be used to interconnect Virtual Private Clouds (VPCs) and on-premises networks. 
As cloud infrastructure expands globally, inter-region peering uses AWS global infrastructure to link Transit Gateways together. 
Data is automatically encrypted and does not traverse the public internet.

## Resource Setting Values

-   `dns_support` : Enable DNS verification for connected VPCs - `enable`, `disable`
-   `vpn_ecmp_support` : ECMP routing support for connected VPNs - `enable`, `disable`
-   `default_route_table_association` : Automatically associate attachments with the main route table - `enable`, `disable`
-   `default_route_table_propagation` : Enable auto propagation of attachments with the main route table - `enable`, `disable`
-   `multicast_support` : Enable create multicast domain - `enable`, `disable`
-   `transit_gateway_cidr_blocks` : Cidr blocks of Transit Gateway
-   `vpc_attachment`
