## Resource Overview

The Transit Gateway is a network transit hub that can be used to interconnect Virtual Private Clouds (VPCs) and on-premises networks. 
As cloud infrastructure expands globally, inter-region peering uses AWS global infrastructure to link Transit Gateways together. 
Data is automatically encrypted and does not traverse the public internet.

## Resource Configuration Values

-   `dns_support` : Enable or disable Domain Name System (DNS) resolution for the attached VPC - `enable`, `disable`
-   `vpn_ecmp_support` : Enable or disable Equal-Cost Multi-Path (ECMP) routing for the attached VPN - `enable`, `disable`
-   `default_route_table_association` : Enable or disable automatic association of the attachment with the default route table - `enable`, `disable`
-   `default_route_table_propagation` : Enable or disable automatic propagation of the attachment to the default route table - `enable`, `disable`
-   `multicast_support` : Enable or disable multicast domain creation - `enable`, `disable`
-   `transit_gateway_cidr_blocks` : CIDR blocks for the Transit Gateway
-   `vpc_attachment`
