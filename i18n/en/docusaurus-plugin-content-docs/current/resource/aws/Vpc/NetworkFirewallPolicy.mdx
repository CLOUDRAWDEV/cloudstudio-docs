## Resource Overview

The AWS Network Firewall Policy defines the monitoring and protection actions for the Network Firewall.

## Resource Configuration Values

-   `description` : Description of the firewall policy
-   `stream_exception_policy` : How to handle midstream traffic interruptions - `drop`, `continue`, `reject`
-   `stateless_default_action` : Stateless default action setting
-   `stateless_default_action.enabled_fragment_packet_action` : Whether fragment packet actions are enabled - `false`, `true`
-   `stateless_default_action.default_action` : Actions to take when a packet does not match any stateless rule in the policy - `aws:drop`, `aws:pass`, `aws:forward_to_sfe`
-   `stateless_rule_group_names` : Stateless rule groups used in the policy
-   `stateful_default_action` : Configuration of the stateful default action
-   `stateful_default_action.rule_evaluation_order` : How to manage the evaluation order of stateful rules in the policy
-   `stateful_default_action.default_actions` : Actions to take when a packet does not match any stateful rule in the policy - `aws:drop_strict`, `aws:drop_established`, `aws:alert_strict`, `aws:alert_established`
-   `strict_stateful_rule_group_names` : Rule groups to apply when a packet does not match any stateful rule
-   `rule_variables` : Variables used to override default rule data settings in the firewall policy
-   `encryption_configuration.enabled_custom_configuration` : Whether to encrypt data using AWS-managed keys - `false`, `true`
