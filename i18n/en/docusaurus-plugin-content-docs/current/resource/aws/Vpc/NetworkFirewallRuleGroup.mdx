## Resource Overview

A reusable set of criteria for inspecting and handling network traffic

## Resource Configuration Values

-   `description`: Description of the network firewall rule group
-   `rule_group_type`: Type of rule group - `STATEFUL`, `STATELESS`
-   `generated_rules_type`: Type of rule group - `StandardStatefulRules`, `DomainList`, `SuricataRuleStrings`
-   `rule_evaluation_order`: Rule evaluation order - `STRICT_ORDER`, `DEFAULT_ACTION_ORDER`
-   `capacity`: Expected number of rules to be included in this rule group over its lifecycle - `1-30000`
-   `ip_set`: IP set variable
-   `ip_set.name`: Name of the IP set variable
-   `ip_set.cidrs`: Values of the IP set variable
-   `port_set`: Port set variable
-   `port_set.name`: Name of the port set variable
-   `port_set.cidrs`: Values of the port set variable
-   `stateful_rule`: Standard stateful rule
-   `stateful_rule.name`: Name of the standard stateful rule
-   `stateful_rule.protocol`: Transport protocol to inspect - `IP`, `TCP`, `UDP`, `ICMP`, `HTTP`
-   `stateful_rule.source_ip_or_cidr`: Source IP address and address range to inspect
-   `stateful_rule.source_port`: Source port or port range to inspect
-   `stateful_rule.target_ip_or_cidr`: Target IP address and address range to inspect
-   `stateful_rule.traffic_direction`: Traffic direction to inspect - `ANY`, `FORWARD`
-   `stateful_rule.action`: Action for the network firewall to take when a packet matches the rule - `ALERT`, `DROP`, `PASS`, `REJECT`
-   `encryption_configuration.enabled_custom_configuration`: Whether to encrypt data using AWS managed keys - `false`, `true`
