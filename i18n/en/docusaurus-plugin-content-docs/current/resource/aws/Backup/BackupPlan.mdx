## Resource Overview

This is a policy expression that defines when and how to back up AWS resources. 
You can assign resources to a Backup Plan, and AWS Backup will automatically create and manage backups for those resources according to the Backup Plan.

### Parent Resource

-   [**Region**](https://docs.cstudio.app/docs/resource/aws/Base/Region)

## Resource Configuration Values

-   `rule.rule_name`: Name of the backup rule
-   `rule.backup_vault_name`: Name of the logical container used for backup storage
-   `rule.frequency_type`: Timing of the backup frequency
-   `rule.frequency_hour`: Hour to initialize the backup plan - `1 or more`
-   `rule.start_time`: Start time of the backup plan
-   `rule.start_after_minutes`: Total time until the backup starts - `60 or more`
-   `rule.complete_after_minutes`: Time (in minutes) AWS Backup will attempt to back up before canceling the job and returning an error - `121 or more`
-   `rule.enabled_use_cold_storage`: Whether to use cold storage
-   `rule.warm_storage_retention_days`: Number of days after creation to move recovery points to cold storage - `1 ~ 36410`
-   `rule.cold_retention_days`: Number of days after creation to delete recovery points - `91 ~ 36500`

-   `resource_assignment.name`
-   `resource_assignment.iam_role_arn`: ARN of the IAM role used by AWS Backup for authentication when restoring and backing up the target resource
-   `resource_assignment.enabled_contain_all`: Whether to include all resources - `false`, `true`
