## Resource Overview

This is a policy expression that defines when and how to back up AWS resources. 
You can assign resources to a Backup Plan, and AWS Backup will automatically create and manage backups for those resources according to the Backup Plan.

### Parent Resource

-   [**Region**](https://docs.cstudio.app/en/docs/resource/aws/Base/Region)

## Associated Resources

## Resource Setting Values

-   `rule.rule_name` : The name of Backup Plan's rule
-   `rule.backup_vault_name` : The name of a logical container where backups are stored
-   `rule.frequency_type` : The frequency timing about back up
-   `rule.frequency_hour` : Determine the exact hour to initiate back up plan - `1 or more`
-   `rule.start_time` : The time to start initiating back up plan
-   `rule.start_after_minutes` : The amount of time in minutes before beginning a backup - `60 or more`
-   `rule.complete_after_minutes` : The amount of time in minutes AWS Backup attempts a backup before canceling the job and returning an error - `121 or more`
-   `rule.enabled_use_cold_storage` : Whether to use cold storage
-   `rule.warm_storage_retention_days` : Specifies the number of days after creation that a recovery point is moved to cold storage - `1 ~ 36410`
-   `rule.cold_retention_days` : Number of days after creation to delete recovery points - `91 ~ 36500`

-   `resource_assignment.name`
-   `resource_assignment.iam_role_arn` : The ARN of the IAM role that AWS Backup uses to authenticate when restoring and backing up the target resource
-   `resource_assignment.enabled_contain_all` : Whether to contain all resources - `false`, `true`
