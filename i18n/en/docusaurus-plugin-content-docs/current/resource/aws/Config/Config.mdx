## Resource Overview

A resource that records the configuration information of resources in an AWS account to identify relationships between them.

## Resource Configuration Values

-   `enable_config_recorder`: Whether to enable or disable the configuration recorder - `false`, `true`
-   `role_name`: The name of the IAM role used by AWS Config
-   `recording_method`
-   `recording_method.enable_all_supported`: Whether AWS Config records configuration changes for all supported regional resource types - `false`, `true`
-   `recording_method.exclude_resource_types`: A list specifying the AWS resource types for which AWS Config should not record configuration changes
-   `recording_method.include_global_resource_types`: Whether AWS Config includes all types of global resources in the recording - `false`, `true`
-   `linked_s3_bucket_name`: The name of the S3 bucket used to store configuration records
-   `s3_key_prefix`: The prefix for the S3 bucket
-   `enable_sns_topic`: Whether to enable the SNS topic for AWS Config notifications
-   `delivery_frequency`: The frequency at which AWS Config sends configuration snapshots
-   `config_rule`
-   `config_rule.name`: The name of the config rule
-   `config_rule.rule_owner`: Whether the AWS Config rule is owned and managed by AWS or by the customer - `AWS`, `customLambda`, `customPolicy`
-   `config_rule.source_identifier`: A predefined identifier for AWS Config managed rules
