## Resource Overview

In API Gateway, HTTP API is a type of API Gateway designed for building APIs based on the HTTP protocol in a simple and lightweight manner. 
HTTP APIs are simpler and more cost-effective than traditional REST APIs, making them ideal for serverless web applications and simple microservices.

## Resource Configuration Values

-   `integration`: Name of the API Gateway Integration
    -   `http_integration`
        -   `integration_type`: Type of API Gateway Integration - `Http`, `Lambda`
        -   `integration_method`: HTTP method for the API Gateway Integration
        -   `integration_url`: URL to be specified for HTTP integration
    -   `lambda_integration`
        -   `integration_type`: Type of API Gateway Integration - `Http`, `Lambda`
        -   `integration_method`: HTTP method for the API Gateway Integration
        -   `linked_lambda_name`: Name of the Lambda function for Lambda proxy integration
-   `route`
    -   `route_method`: HTTP method for the HTTP API
    -   `route_path`: Resource path for the HTTP API
    -   `target_integration`: Name of the API Gateway Integration to link to the route - `http_integration`, `lambda_integration`
-   `deployment`
    -   `description`: Description of the API Gateway deployment
-   `stage`
    -   `dev`
        -   `enable_deployment`: Enable deployment for the API Gateway stage
        -   `auto_deployment`: Enable automatic deployment
    -   `test`
        -   `enable_deployment`: Enable deployment for the API Gateway stage
        -   `auto_deployment`: Enable automatic deployment
        -   `deployment_name`: Name of the API Gateway deployment to connect
    -   `prod`
        -   `enable_deployment`: Enable deployment for the API Gateway stage
