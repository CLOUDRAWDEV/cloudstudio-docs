## Resource Overview

Amazon Elasticsearch Service (Amazon ES) is a distributed search and analytics engine designed for fast search and data analysis. 
Built on top of the Apache Lucene search engine, Elasticsearch is a fast, scalable open-source solution. 
It is widely used for data collection, log analysis, and monitoring. 
Amazon Elasticsearch Service can run within an Amazon VPC and integrates seamlessly with AWS services like Lambda and Kinesis. It also offers robust security and access control features provided by AWS, and integrates with Amazon CloudWatch for monitoring and logging capabilities.

## Resource Setting Values

-   `elasticsearch_version`: The version of the Elasticsearch domain.
-   `availability_zone_count`: The number of availability zones - `2`, `3`.
-   `tls_security_policy`: The name of the TLS security policy to apply to the HTTPS endpoint - `Policy-Min-TLS-1-2-2019-07`, `Policy-Min-TLS-1-0-2019-07`.
-   `data_node`
    -   `instance_type`: The instance type of the data nodes in the cluster.
    -   `instance_count`: The number of instances in the cluster - `1`~`80`.
    -   `ebs_enabled`: Whether EBS volumes are attached to the domain's data nodes - `true`, `false`.
    -   `ebs_volume_type`: The type of EBS volume attached to the data nodes - `standard`, `gp2`, `io1`.
    -   `ebs_volume_size`: The size of the EBS volumes attached to the data nodes (GiB) - `1`~`16384`.
-   `master_node`
    -   `master_node_enabled`: Whether a dedicated master node is enabled in the cluster - `true`, `false`.
    -   `master_instance_type`: The instance type of the dedicated master nodes in the cluster - `r5.large.search`.
    -   `master_node_count`: The number of dedicated master nodes in the cluster - `3`, `5`.
-   `custom_endpoint`
    -   `custom_endpoint_enabled`: Whether a custom endpoint is enabled - `true`, `false`.
-   `network`
    -   `network_type`: The type of network access - `vpc_access`, `public_access`.
    -   `included_subnet_names`: A list of VPC subnet IDs for the Elasticsearch domain endpoint.
    -   `security_group_names`: A list of VPC security group IDs to apply to the Elasticsearch domain endpoint.
-   `fine_grained_access`
    -   `fine_grained_access_enabled`: Whether fine-grained access control is enabled - `true`, `false`.
    -   `master_user_type`: The type of master user - `iam_arn`, `master_user`.
    -   `iam_arn`: The IAM ARN for the master user.
-   `encryption`
    -   `enforce_https`: Whether HTTPS is required - `true`, `false`.
    -   `node_to_node_encryption_enabled`: Whether node-to-node encryption is enabled - `true`, `false`.
    -   `data_at_rest_encryption_enabled`: Whether encryption at rest is enabled for the data - `true`, `false`.
-   `kms_key_name`: The name of the KMS key to use for data encryption.
