## Resource Setting Values

-   `set_container_insights`: Whether to enable Container Insights for the cluster - `Enabled`
-   `kms_key_name`: AWS Key Management Service key name for encrypting data between local clients and containers
-   `logging`: Log settings for redirecting command execution results - `OVERRIDE`, `NONE`, `DEFAULT`
-   `cloud_watch_encryption_enabled`: Whether encryption is enabled for CloudWatch logs - `false`, `true`
-   `capacity_providers`: Names of the CloudWatch log groups to send logs to - `Fargate`, `Fargate_spot`
-   `service`:
  -     `service.name`: Name of the service
  -     `service.computing_option`: Method of distributing tasks for the service - `capacity_provider_strategy`, `launch_type`

-   `service.computing_option.capacity_provider_strategy`:
  -     `service.computing_option.capacity_provider_strategy.capacity_provider`: `Fargate`, `Fargate_spot`
  -     `service.computing_option.capacity_provider_strategy.base`: Minimum number of tasks to run (up to 100,000)
  -     `service.computing_option.capacity_provider_strategy.weight`: Relative percentage of the total number of tasks (up to 1,000)

-   `service.computing_option.launch_type`: Launch type for the service
  -     `service.computing_option.launch_type.launch_type`: `ec2`, `Fargate`, `External`
  -     `service.computing_option.launch_type.task_definition_name`: Name of the ECS Task Definition used by the service
  -     `service.computing_option.launch_type.desired_count`: Number of task instances to run for the service
