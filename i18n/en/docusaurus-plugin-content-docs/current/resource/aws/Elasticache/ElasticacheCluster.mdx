## Resource Overview

The AWS ElastiCache Cluster is a cloud-based in-memory data store that provides a fast and scalable caching solution. 
ElastiCache Clusters can run using either the Redis or Memcached engine in cluster mode or single-node mode. 
They are used to enhance application performance by enabling faster data retrieval and processing. 
ElastiCache Clusters offer features such as automated backups, recovery, scaling, and monitoring, ensuring that users can securely store and manage their data with confidence.

## Resource Configuration Values

-   `engine`: The name of the cache engine to be used in the Cache Cluster - `redis`, `memcached`
-   `engine_version`: The version of the cache engine
-   `port`: The port number used to connect to each cache node. Memcached uses port 11211, and Redis uses port 6379 by default
-   `parameter_group_name`: The name of the parameter group to associate with the Cache Cluster
-   `node_type`: The instance class used in the Cache Cluster
-   `num_cache_nodes`: The initial number of cache nodes in the Cache Cluster - `1` to `40`
-   `az_mode`: Specifies whether the Memcached node group is created in a single Availability Zone or across multiple Availability Zones within the cluster's region - `single-az`, `cross-az`
-   `included_subnet_names`: The names of the subnets included in the Subnet Group where the ElastiCache Cluster will be deployed
-   `snapshot_retention_limit`: The period for which ElastiCache retains automatic cache cluster snapshots before deletion
