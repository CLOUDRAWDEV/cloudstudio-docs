## Resource Overview

Amazon Aurora(Aurora)는 MySQL 및 PostgreSQL과 호환되는 완전 관리형 관계형 데이터베이스 엔진입니다. MySQL 및 PostgreSQL이 이 고급 상용 데이터베이스의 속도와 안정성을 오픈 소스 데이터베이스의 단순성 및 비용 효율성과 어떻게 결합하는지 이미 알고 계실 것입니다. 오늘날 기존 MySQL 및 PostgreSQL 데이터베이스에 사용되는 코드, 도구 및 애플리케이션 모두 Aurora에서도 사용할 수 있습니다. 일부 워크로드의 경우 Aurora은 기존 애플리케이션을 거의 변경하지 않고도 MySQL의 처리량을 최대 5배, PostgreSQL의 처리량을 최대 3배 제공할 수 있습니다.

## Resource Configuration Values

-   `database_name` : Initial database name
-   `engine` : Aurora Engine type to use with RDS - `aurora`, `aurora-mysql`, `aurora-postgresql`
-   `engine_version` : Aurora Engine version to use with RDS
-   `instance_class` : Pricing tier for the RDS DB instance
-   `engine_mode` : Aurora engine mode - `aurora`, `multimaster`, `parallelquery`, `serverless`
-   `multi_az_enabled` : Enable multi-AZ deployment - `true`, `false`
-   `master_username` : Master username
-   `master_password` : Master password
-   `included_subnet_names` : Subnet names to connect to the RDS subnet group
-   `subnet_group_name` : Name of the subnet group to use with RDS
-   `monitoring`
    -   `performance_insights_enabled` : Enable Performance Insights
-   `encryption`
    -   `storage_encrypted` : Specify whether the DB instance is encrypted - `true`, `false`
    -   `kms_key_name` : Name of the KMS key to use for encryption
-   `backup`
    -   `backup_retention_period` : Period for retaining backup data - `1` to `35`
    -   `final_snapshot_enabled` : Whether to create a final DB snapshot before deleting the DB instance - `true`, `false`
    -   `final_snapshot_name` : Name of the final snapshot
