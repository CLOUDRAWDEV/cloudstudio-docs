## Resource Overview

Using MySQL with RDS on AWS makes database management and operations easier. 
RDS is highly scalable, allowing you to improve performance by changing instance types as needed. 
Additionally, RDS provides automatic backups and restore capabilities to prevent data loss. 
AWS's security features also ensure the safety and protection of your data.

## Resource Configuration Values

-   `database_name`: Initial database name
-   `engine_version`: MySQL version to be used with RDS
-   `instance_class`: Pricing tier for the RDS DB instance
-   `storage_type`: Storage type for the RDS instance
-   `allocated_storage`: Storage capacity of the RDS instance (in gigabytes)
-   `max_allocated_storage`: Maximum storage limit for automatic scaling by Amazon RDS
-   `multi_az_enabled`: Enable multi-AZ deployment - `true`, `false`
-   `username`: Username for RDS
-   `password`: Password for RDS
-   `skip_final_snapshot`: Whether to create a final DB snapshot before deleting the DB instance - `true`, `false`
-   `included_subnet_names`: Names of the subnets to be included in the RDS subnet group
-   `subnet_group_name`: Name of the subnet group for RDS
-   `publicly_accessible`: Whether the instance is publicly accessible - `true`, `false`

-   `monitoring`
    -   `performance_insights_enabled`: Enable performance insights

-   `encryption`
    -   `storage_encrypted`: Whether the DB instance is encrypted - `true`, `false`
    -   `kms_key_name`: Name of the KMS key to use for encryption

-   `backup`
    -   `backup_retention_period`: Duration to retain backup data - `1` to `35` days
    -   `final_snapshot_enabled`: Whether to create a final DB snapshot before the DB instance is deleted - `true`, `false`
    -   `final_snapshot_name`: Name of the final snapshot
