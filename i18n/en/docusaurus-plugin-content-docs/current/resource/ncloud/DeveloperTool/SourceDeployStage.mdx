## 리소스 설정 값

-   `included_source_deploy_name` : SourceDeploy 프로젝트 이름
-   `target_type` : 배포 대상의 유형 - `Server`,`AutoScalingGroup`,`KubernetesService`,`ObjectStorage`
-   `linked_server_list` : 연결할 서버 이름 리스트
-   `included_subnet_names` : 클러스터를 구성할 서브넷 이름 목록, 최대 3개
-   `scenario` : 배포 환경을 실행할 수 있는 배포 시나리오
    -   `deploy_scenario_description` : source deploy 프로젝트 설명
    -   `deploy_scenario_strategy` : 배포 전략 - `normal`,`blueGreen`,`rolling`,`canary`
    -   `deploy_scenario_file_type` : 배포 시나리오의 파일 타입 - `SourceBuild`,`ObjectStorage`,`later`
    -   `linked_source_build_project` : source build 프로젝트의 이름 
    -   `rollback` : 배포 실패 시 롤백할 지 여부
    -   `pre_deploy_user` : 배포 전 실행 계정
    -   `pre_deploy_command` : 배포 전 실행 명령어
    -   `source_path_command` : 배포할 원본 파일 경로
    -   `deploy_path_command` : 배포할 경로
    -   `post_deploy_user` : 배포 후 실행 계정
    -   `post_deploy_command` : 배포 후 실행 명령어
