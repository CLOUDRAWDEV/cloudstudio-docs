## Resource Overview

This resource provides an environment for conveniently managing Elasticsearch and OpenSearch clusters, allowing you to configure clusters by selecting various server types and storage capacities, and enabling easy deployment, operation, and scaling.

## Resource Setting Values

-   `os_image_code` : The OS type to be used
-   `search_engine` : The main items of Search Engine
    -   `version_code` : The Search Engine Service version to be used
    -   `user_name` : The Search Engine UserName
    -   `user_password` : The Search Engine User password
    -   `dashboard_port` : The Search Engine Dashboard port (1024 ~ 65535)
-   `manager_node` : The main items of manager node
    -   `enable_dual_manager` : Whether to make redundancy of manager node
    -   `manager_node_product_code` : Whether to make redundancy of manager node
    -   `manager_node_subnet_name` : The subnet number where the manager node is to be located
-   `data_node` : The main items of data node
    -   `data_node_product_code` : The HW specifications of the data node
    -   `data_node_subnet_name` : The subnet number where the data node is to be located
    -   `data_node_count` : Number of data nodes (3 ~10)
    -   `data_node_storage_size` : The data node storage capacity (100~2000)
-   `enable_master_node` : Whether to use master node
-   `login_key_name` : The name of Login key to encrypt with the public key
