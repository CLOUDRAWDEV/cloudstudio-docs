## 리소스 개요

Amazon Aurora(Aurora)는 MySQL 및 PostgreSQL과 호환되는 완전 관리형 관계형 데이터베이스 엔진입니다. MySQL 및 PostgreSQL이 이 고급 상용 데이터베이스의 속도와 안정성을 오픈 소스 데이터베이스의 단순성 및 비용 효율성과 어떻게 결합하는지 이미 알고 계실 것입니다. 오늘날 기존 MySQL 및 PostgreSQL 데이터베이스에 사용되는 코드, 도구 및 애플리케이션 모두 Aurora에서도 사용할 수 있습니다. 일부 워크로드의 경우 Aurora은 기존 애플리케이션을 거의 변경하지 않고도 MySQL의 처리량을 최대 5배, PostgreSQL의 처리량을 최대 3배 제공할 수 있습니다.

## 리소스 설정 값

-   `database_name` : 초기 데이터베이스 이름
-   `engine` : RDS에 사용할 Aurora Engine 종류 - `aurora`, `aurora-mysql`, `aurora-postgresql`
-   `engine_version` : RDS에 사용할 Aurora Engine 버전
-   `instance_class` : RDS DB의 가격 측정 계층
-   `engine_mode` : `aurora`, `multimaster`, `parallelquery`, `serverless`
-   `multi_az_enabled` : 다중 AZ 배포 활성화 - `true`, `false`
-   `master_username` : 마스터 사용자 이름
-   `master_password` : 마스터 사용자 비밀번호
-   `included_subnet_names` : RDS Subnet 그룹에 연결할 Subnet 이름
-   `subnet_group_name` : RDS에 사용할 Subnet 그룹 이름
-   `monitoring`
    -   `performance_insights_enabled` : 성능 인사이트 활성화
-   `encryption`

    -   `storage_encrypted` : DB 인스턴스의 암호화 여부를 지정 - `true`, `false`
    -   `kms_key_name` : 암호화에 사용할 kms 키 이름

-   `backup`
    -   `backup_retention_period` : 백업 데이터를 보관할 기간 - `1`~`35`
    -   `final_snapshot_enabled` : DB 인스턴스가 삭제되기 전, 최종 DB 스냅샷을 생성할지 여부 - `true`, `false`
    -   `final_snapshot_name` : 최종 스냅샷 이름
