## リソース概要

ユーザーが決めたスケジュールやモニタリング結果に応じて仮想サーバーの数を自動的に増減し、需要の変化に弾力的に対応するサービスです。
あらかじめ指定しておいたしきい値に基づいてサーバーをモニタリングしながらサーバー数を自動的に調節してサービスを安定的に運営することができます。

## リソース設定値

-   `server_image_product_code` : 使用するオペレーティング システム イメージ コード(CNTOS:CentOS, UBNTU:Ubuntu, WND:Windows / MYSQL,MARIA:Database / SVR:Server, EN:English Edition)
-   `server_product_code` : サーバのハードウェア構成のためのコード(HICPU:High CPU, STAND:Standard, HIMEM:High Memory / C コア数,M メモリ(GB))
-   `login_key_name` : 暗号化用のパブリックログインキー名
-   `init_script`
    -   `content` : サーバーが生成されたときに自動的に実行するスクリプトの内容
-   `group`
    -   `min_size` : Auto scaling グループの最小サイズ、0 ~ 30 の範囲
    -   `max_size` : Auto scaling グループの最大サイズ、0 ~ 30 の範囲
    -   `linked_acg_names` : Auto scaling に適用する ACG の名前リスト
    -   `included_subnet_name` : サーバーを生成するためのサブネット名
-   `policy`
    -   `adjustment_type_code` : Scaling 設定コード、増減変更(CHANG)/比率変更(PRCNT)/固定値(EXACT) - `CHANG`, `PRCNT`, `EXACT`
    -   `scaling_adjustment` : Scaling 設定値
-   `schedule`
    -   `min_size` : Auto scaling グループの最小サイズ
    -   `max_size` : Auto scaling グループの最大サイズ
    -   `desired_capacity` : Auto scaling グループの期待サイズ
    -   `start_time` : スケジュールの開始日および時間
    -   `end_time` : スケジュールの終了日および時間
