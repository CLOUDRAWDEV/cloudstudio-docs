## リソース概要

Cloud DB for Redis は Redis キャッシュをいくつかの設定とクリックだけで簡単に構築し、NAVER Cloud の最適化設定を通じて安定的に運営し、障害発生時に自動復旧する完全管理型クラウドキャッシュサービスです。

## リソース設定値

-   `server_name_prefix` : Redis Cache サーバ名
-   `redis_mode` : Redis の構成モードの種類 - `CLUSTER`, `SIMPLE`
-   `image_product_code` : Redis 構成インスタンススペックで生成しようとする redis サーバーイメージ製品コード
-   `target_config_group_name` : 接続する Config Group
-   `product_code` : 生成する Redis インスタンスのサーバー仕様を決定する製品コード
-   `data_storage_type` : データストレージの種類 - `SSD`, `HDD`
-   `shard_count` : 生成するシャード数。master node の数 - `3~10`
-   `shard_copy_count` : master node と slave node で構成されている Redis cluster のシャード当たりのレプリカの数、HA のためには slave node が必須。 レプリカを追加すると、すべての master node にそれぞれ 1 個ずつ slave node が割り当てられます
-   `enable_high_availability` : 高可用性機能の使用有無
-   `enable_backup` : バックアップ機能を使用するかどうか、もし高可用性機能を使用すると、バックアップ機能は使用によって固定
-   `port` : Redis 接続ポート - `6379`
