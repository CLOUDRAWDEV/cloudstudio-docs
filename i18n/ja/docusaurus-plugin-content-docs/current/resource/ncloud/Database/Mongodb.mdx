## リソース概要

Cloud DB for MongoDB は MongoDB データベースを簡単に構築し、自動的に管理することができます。
NAVER Cloud の最適化設定を通じて安定的に運営し、障害発生時に自動復旧する完全管理型クラウドデータベースサービスです。

## リソース設定値

-   `server_name_prefix` : MongoDB サーバー名
-   `user_name` : MongoDB ユーザーアクセス ID
-   `user_password` : MongoDB のユーザーパスワード
-   `cluster_type_code` : MongoDB のクラスタタイプを決定する MongoDB クラスタタイプコード - `Stand Alone`, `Stand Replica Set`, `Shared Cluster`
-   `image_product_code` : MongoDB バージョンのイメージ製品番号
-   `member_port` : MongoDB Member サーバーにアクセスできる TCP ポート番号 - `10000-65535`
-   `member_server_count` : MongoDB Member サーバーの数、1 つのレプリカセットあたりのメンバーサーバーの数が決まります - `3-7`
-   `mongos_server_count` : MongoDB Mongos サーバーの数 - `2-5`
-   `backup_file_retention_period` : バックアップファイルの保管期間、バックアップは日単位で行われ、バックアップファイルは別のストレージに保存 - `1~30`
-   `data_storage_type` : データストレージの種類 - `SSD`, `HDD`
-   `compress_code` : MongoDB が提供するデータ圧縮アルゴリズムを選択できる MongoDB データ圧縮アルゴリズムコード - `SNPP`, `ZLIB`, `ZSTD`, `NONE`
