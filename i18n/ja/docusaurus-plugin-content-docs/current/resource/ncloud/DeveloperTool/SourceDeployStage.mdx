## リソース概要

生成した SourceDeploy プロジェクトに DeployStage を追加/修正/削除したり、デプロイターゲットがないデプロイ Stage にデプロイターゲットを生成することができます。

## リソース設定値

-   `included_source_deploy_name` : SourceDeploy プロジェクト名
-   `target_type` : デプロイ対象のタイプ - `Server`,`AutoScalingGroup`,`KubernetesService`,`ObjectStorage`
-   `linked_server_list` : 接続するサーバー名のリスト
-   `included_subnet_names` : クラスターを構成するサブネット名のリスト、最大 3 つ
-   `scenario` : デプロイ環境を実行できる配布シナリオ
    -   `deploy_scenario_description` : Source deploy プロジェクトの説明
    -   `deploy_scenario_strategy` : デプロイ戦略 - `normal`,`blueGreen`,`rolling`,`canary`
    -   `deploy_scenario_file_type` : デプロイシナリオのファイルタイプ - `SourceBuild`,`ObjectStorage`,`later`
    -   `linked_source_build_project` : Source build プロジェクトの名前
    -   `rollback` : デプロイに失敗した場合、ロールバックするかどうかの可否
    -   `pre_deploy_user` : デプロイ前の実行アカウント
    -   `pre_deploy_command` : デプロイ前の実行コマンド
    -   `source_path_command` : デプロイする元のファイルパス
    -   `deploy_path_command` : デプロイするパス
    -   `post_deploy_user` : デプロイ後の実行アカウント
    -   `post_deploy_command` : デプロイ後の実行コマンド
