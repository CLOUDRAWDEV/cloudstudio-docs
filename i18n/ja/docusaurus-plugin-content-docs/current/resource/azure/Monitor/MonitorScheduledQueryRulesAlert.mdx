## リソース概要

Azure Monitor 예약된 쿼리 규칙 경고는 사용자가 사용자 지정 로그 쿼리를 기반으로 경고를 만들 수 있도록 하는 Azure Monitor 내의 기능입니다.
예약된 쿼리 규칙(SQR)을 통해 사용자는 설정된 일정에 따라 실행되는 쿼리를 정의한 다음 지정된 임계값 또는 기준에 따라 결과를 평가할 수 있습니다.
쿼리 결과가 정의된 기준을 충족하면 SQR은 이메일, SMS, 웹후크 또는 Azure Functions와 같은 다양한 대상으로 라우팅할 수 있는 경고를 트리거합니다.
고에는 경고를 트리거한 특정 쿼리 결과에 대한 정보와 관련 메타데이터 또는 컨텍스트 정보가 포함될 수 있습니다.

## リソース設定値

-   `linked_application_insights_name` : 로그 검색 쿼리가 실행될 리소스 URI

-   `severity` : 경고의 심각도 - `0`, ` 1````,  `2` , ```3 `, `4`

-   `frequency` : 경고 규칙이 실행되는 빈도 - `5`~ ```1440````

-   `time_window` : 쿼리를 위해 데이터를 수집하는 시간 범위(분) 5 ~ 2880의 범위 - `5`, ```2880````

-   `action_group_name` : 경고 규칙을 적용할 Action Group 이름

-   `email_subject` : Action Group의 모든 이메일에 적용할 제목 재정의

-   `custom_webhook_payload` : 경고 작업에서 모든 웹훅 페이로드에 대해 보낼 사용자 지정 페이로드

-   `operator` : 메트릭 값과 임계 값을 비교하기 위해 사용하는 연산자 - `Equal`, ` GreaterThan````,  `LessThan````

-   `threshold` : 경고를 활성화하는 기준 임계값 - `0`~ ```10000````

-   `tag` : 리소스를 카테고리로 분류하기 위한 태그 - 최대 512자, `key`:`value`
