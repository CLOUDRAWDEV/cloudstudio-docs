## リソース概要

Azure Monitor 自動サイズ調整は、あらかじめ定義されたルール及びメトリックに基づいてリソースの自動サイズ調整を可能にする Microsoft Azure の機能です。
自動サイズ調整を使用して仮想マシン、アプリサービス及び AKS(Azure Kubernetes Service) クラスターなどのリソースを拡張または縮小して、変化する要件を満たし、コストを最適化することができます。

## リソース設定値

-   `target_resource_name` : 自動サイズ調整設定を適用するリソース名

-   `scale_condition`

    -   `min_instance_count` : 最小インスタンス数 - `0` ~ `1000`
    -   `max_instance_count` : 最大インスタンス数 - `1` ~ `1000`
    -   `default_instance_count` : リソースメトリックが読み取れず、現在の容量がデフォルトの容量未満の場合、スケールアウトするデフォルトインスタンスの数 - `1` ~ `1000`
    -   `schedule_type` : サイズ調整作業に適用するスケジュール方式 - `start_end_dates`, `repeat_days`
    -   `timezone` : スケジュールが働く時間帯
    -   `start_date` : スケジュール開始日
    -   `end_date` : スケジュールが終わる日

-   `tag` : リソースをカテゴリに分類するためのタグ - 最大 512 文字, `key`:`value`
