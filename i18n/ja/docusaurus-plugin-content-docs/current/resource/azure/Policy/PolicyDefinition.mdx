## リソース概要

Azure ポリシー定義はガバナンスポリシーを適用し監査する方法を提供します。ポリシー定義を使用してリソースプロパティ、タグ、またはリソースグループのルールを定義できます。
たとえば、すべての仮想マシンが特定のタグを持つように要求したり、特定の地域での仮想マシンの生成を制限するポリシー定義を作成したりできます。
ポリシー定義を作成した後は、リソース グループまたはサブスクリプションなどの Azure の範囲に割り当てることができます。これにより、その範囲内のすべてのリソースにポリシーが適用され、リソースがガバナンスポリシーに準拠していることを確認するためにコンプライアンスレポートが提供されます。

## リソース設定値

-   `type` : ポリシーのタイプ - `Custom`,`BuiltIn`,`NotSpecified`

-   `mode` : 評価するリソースタイプを指定するポリシーモード

-   `display_name` : ポリシー定義の表示名、1 ~ 128 文字

-   `linked_resource_group_name` : Policy を適用する Resource Group の MyPageID と名前
