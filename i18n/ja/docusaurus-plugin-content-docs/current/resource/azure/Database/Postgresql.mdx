## リソース概要

Azure Database for PostgreSQL は広く使用されているオープンソース PostgreSQL データベースエンジンを基盤とする Microsoft Azure が提供する完全に管理されたリレーショナルデータベースサービスです。
基本インフラについて心配することなく、クラウド上で MySQL データベースを簡単にデプロイ、管理、拡張することができます。
Azure Database for PostgreSQL を使用すると、 Microsoft がパッチ適用、バックアップ、モニタリングなどの作業を処理するので、最小限の管理オーバーヘッドで MySQL データベースを作成·管理することができます。
また、高可用性、自動バックアップ、および指定時間復元などの機能を活用することもできます。

## リソース設定値

-   `version` : 使用する PostgreSQL のバージョン - `9.5`, ` 9.6``,  `10` , ```10.0 `, ` 10.2``,  `11``

-   `sku_name` : PostgreSQL 価格設定階層名、Tier+コンピューティング世代+コア数の組み合わせ - `azureDatabaseSku`

-   `storage_mb` : PostgreSQL Server の最大容量(MB) - `5120`~`16777216`

-   `create_mode` : MySQL Flexible Server のストレージ IOPS - `Default`,`Replica`,`GeoRestore`,`PointInTimeRestore`

-   `admin_username` : Creation mode がデフォルトの場合、サーバーの管理者ユーザー名

-   `admin_password` : Creation mode がデフォルトの場合、サーバーの管理者ログインパスワード

-   `public_network_access_enabled` : サーバーへのパブリックネットワークへのアクセスを許容 - `true` , `false`

-   `ssl_enforcement_enabled` : 暗号化された接続(SSL)適用の可否 - `true` , `false`

-   `minimal_tls_version` : サーバーにサポートされる最小 TLS バージョン - `TLS1_2` , `TLS1_1` , `TLS1_0` , `TLSEnforcementDisabled`

-   `database`

    -   `charset` : PostgreSQL データベースの文字セット - `azurePostgresqlChartSet`
    -   `collation` : PostgreSQL Database のデータ整列方式 - `azurePostgresqlCol`

-   `firewall_rule`

    -   `start_ip_address` : Firewall Rule に接続された開始 IP アドレス
    -   `end_ip_address` : Firewall Rule に接続された終了 IP アドレス

-   `tag` : リソースをカテゴリに分類するためのタグ - 最大 512 文字, `key`:`value`
