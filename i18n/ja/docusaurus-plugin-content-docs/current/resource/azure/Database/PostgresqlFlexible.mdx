## リソース概要

Azure Database for PostgreSQL 柔軟なサーバは柔軟なサーバアーキテクチャを提供する Azure の拡張可能な完全管理型オープンソース関係型データベースサービスです。
PostgreSQL 環境構成に対するより多くの制御が必要な様々なワークロードおよびアプリケーションをサポートするように設計されました。
柔軟なサーバーにより、お客様は特定の要件を満たすために、さまざまなコンピューティングおよびストレージ オプションから選択できます。
単一ノードと高可用性構成の両方をサポートし、バックアップと災害復旧のための複数のオプションを提供します。

## リソース設定値

-   `version` : 使用する PostgreSQL のバージョン - `11`, `12`, `13`, `14`

-   `sku_name` : PostgreSQL 価格設定階層名、Tier+コンピューティング世代+コア数の組み合わせ - `azurePostgresqlFlexibleSku`

-   `storage_mb` : Postgresql Flexible Server の最大容量(MB) - `storageMB`

-   `admin_username` : Creation mode がデフォルトの場合、サーバーの管理者ユーザー名

-   `admin_password` : Creation mode がデフォルトの場合、サーバーの管理者ログインパスワード

-   `availability_zone` : アベイラビリティ ゾーンを指定します。 指定できる値は 1、2、3 です。 - `1` ~ `3`

-   `enable_high_availability` : Postgresql Flexible Server の高可用性活性化の可否 - `true` , `false`

-   `backup_retention_days` : Postgresql Flexible Server のバックアップ保存日数です。可能な値は 1 ~ 35 日です。 - `1` ~ `35`

-   `geo_redundant_backup_enabled` - `true` , `false`

-   `public_network_access_enabled` : Server へのパブリックネットワークへのアクセスを許容 - `true` , `false`

-   `configuration`

    -   `name` : Postgresql Flexible Server Configuration の名前
    -   `param_value` : Postgresql Flexible Server Configuration の値

-   `database`

    -   `charset` : PostgreSQL データベースの文字セット - `azurePostgresqlChartSet`
    -   `collation` : PostgreSQL Database のデータ整列方式 - `azurePostgresqlFlexibleCol`

-   `firewall_rule`

    -   `start_ip_address` : Firewall Rule に接続された開始 IP アドレス
    -   `end_ip_address` : Firewall Rule に接続された終了 IP アドレス

-   `tag` : リソースをカテゴリに分類するためのタグ - 最大 512 文字, `key`:`value`
