## 리소스 개요

Azure Database for MySQL은 널리 사용되는 오픈 소스 MySQL 데이터베이스 엔진을 기반으로 하는 Microsoft Azure에서 제공하는 완전히 관리되는 관계형 데이터베이스 서비스입니다.
기본 인프라에 대해 걱정할 필요 없이 클라우드에서 MySQL 데이터베이스를 쉽게 배포, 관리 및 확장할 수 있습니다.
Azure Database for MySQL을 사용하면 Microsoft가 패치 적용, 백업 및 모니터링과 같은 작업을 처리하므로 최소한의 관리 오버헤드로 MySQL 데이터베이스를 만들고 관리할 수 있습니다.
또한 고가용성, 자동 백업 및 지정 시간 복원과 같은 기능을 활용할 수도 있습니다.

## 리소스 설정 값

-   `version` : 사용할 MySQL 버전 - `5.7`, `5.6`, `8.0`

-   `sku_name` : MySQL 가격 책정 계층 이름, Tier + 컴퓨팅 세대 + 코어 수의 조합 - `azureDatabaseSku`

-   `storage_mb` : MySQL Server의 용량(MB) - `5120` ~ `16777216`

-   `admin_username` : Creation mode가 default인 경우 Server의 관리자 사용자 이름

-   `admin_password` : Creation mode가 default인 경우 Server의 관리자 로그인 암호

-   `public_network_access_enabled` : Server에 퍼블릭 네트워크 접근 허용 - `true` , `false`

-   `ssl_enforcement_enabled` : 암호화된 연결(SSL) 적용 여부 - `true` , `false`

-   `minimal_tls_version` : Server에 지원되는 최소 TLS 버전 - `TLS1_2` , `TLS1_1` , `TLS1_0` , `TLSEnforcementDisabled`

-   `threat_detection_enabled` : 비정상적이고 잠재적으로 유해한 시도를 나타내는 비정상적인 활동 감지 여부 - `true` , `false`

-   `database`

    -   `charset` : MySQL Database의 문자 집합 - `azureMysqlChartSet`
    -   `collation` : Microsoft SQL 데이터베이스의 데이터 정렬 방식 - `azureMysqlCol`

-   `firewall_rule`
    -   `start_ip_address` : 방화벽 Rule에 연결된 시작 IP 주소
    -   `end_ip_address` : 방화벽 Rule에 연결된 종료 IP 주소
-   `tag` : 리소스를 카테고리로 분류하기 위한 태그 - 최대 512자, `key`:`value`
