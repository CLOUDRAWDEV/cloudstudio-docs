## リソース概要

Azure Database for MySQL は広く使用されているオープンソース MySQL データベースエンジンを基盤とする Microsoft Azure が提供する完全に管理されたリレーショナルデータベースサービスです。
基本インフラについて心配することなく、クラウド上で MySQL データベースを簡単にデプロイ、管理、拡張することができます。
Azure Database for MySQL を使用すると、Microsoft がパッチ適用、バックアップ、モニタリングなどの作業を処理するので、最小限の管理オーバーヘッドで MySQL データベースを作成·管理することができます。
また、高可用性、自動バックアップ、および指定時間復元などの機能を活用することもできます。

## リソース設定値

-   `version` : 使用する MySQL のバージョン - `5.7`, `5.6`, `8.0`

-   `sku_name` : MySQL 価格設定階層名、Tier+コンピューティング世代+コア数の組み合わせ - `azureDatabaseSku`

-   `storage_mb` : MySQL Server の容量(MB) - `5120` ~ `16777216`

-   `admin_username` : Creation mode がデフォルトの場合、サーバーの管理者ユーザー名、英数字の許容、予約語を含まない、1~16 文字

-   `admin_password` : Creation mode がデフォルトの場合、サーバーの管理者ログインパスワード、小文字/大文字/数字/特殊文字のうち 3 つを含む、ログイン名から 3 つ以上の連続した英数字を含めることができない、8~128 文字

-   `public_network_access_enabled` : サーバーへのパブリック ネットワークへのアクセスを許容 - `true` , `false`

-   `ssl_enforcement_enabled` : 暗号化された接続(SSL)適用の可否 - `true` , `false`

-   `minimal_tls_version` : サーバーにサポートされる最小 TLS バージョン - `TLS1_2` , `TLS1_1` , `TLS1_0` , `TLSEnforcementDisabled`

-   `threat_detection_enabled` : 異常で有害な可能性のある試行を示す異常なアクティビティの検出を有効にする - `true` , `false`

-   `database`

    -   `charset` : MySQL データベースの文字セット - `azureMysqlChartSet`
    -   `collation` : MySQL Database のデータ整列方式 - `azureMysqlCol`

-   `firewall_rule`
    -   `start_ip_address` : Firewall Rule に接続された開始 IP アドレス
    -   `end_ip_address` : Firewall Rule に接続された終了 IP アドレス
-   `tag` : リソースをカテゴリに分類するためのタグ - 最大 512 文字, `key`:`value`
