## リソース概要

Azure SQL Database は広く使用されている Microsoft SQL Server エンジンを基盤とする Microsoft Azure が提供する完全に管理された関係型データベースサービスです。
ハードウェアまたはソフトウェアインフラを管理する必要なく、クラウドで SQL Server データベースを作成して管理することができます。
Azure SQL データベースを使用すると、自動パッチ、バックアップ、高可用性、および災害復旧などの機能を活用できます。
また、インテリジェントなパフォーマンス監視、自動調整、高度なセキュリティ機能などの機能を提供することで、データベースのパフォーマンスを最適化し、重要なデータを保護できます。

## リソース設定値

-   `version` : 使用する Microsoft SQL サーバーのバージョン - `2.0`, `12.0`

-   `admin_username` : サーバーの管理者ユーザー名

-   `admin_password` : サーバーの管理者ログインパスワード

-   `minimum_tls_version` : サーバーにサポートされる最小 TLS バージョン - `1.0`, `1.1`, `1.2`

-   `public_network_access_enabled` : サーバーへのパブリック ネットワークへのアクセスを許許容 - `true`, `false`

-   `firewall_rule`

    -   `start_ip_address` : Firewall Rule に接続された開始 IP アドレス
    -   `end_ip_address` : Firewall Rule に接続された終了 IP アドレス

-   `security_alert_policy`

    -   `alert_policy_state` : 適用されるポリシーのアクティベーション状態
    -   `disabled_alerts` : 無効にするセキュリティ警告のリスト - `Sql_Injection`, `Sql_Injection_Vulnerability`, `Access_Anomaly`, `Data_Exfiltration`, `Unsafe_Action`
    -   `alert_retention_days` : 脅威検出監査ログの保管日数 - `0` ~ `3265`
    -   `alert_email_account_admins` : 電子メールへの警告の送信を有効にする - `true` , `false`
    -   `alert_email_addresses` : セキュリティ警告を受ける電子メールアドレスのリスト

-   `auditing_policy`

    -   `enable_storage_destination` : Storage アカウント活性化の可否 - `true` , `false`
    -   `enable_log_analytics_destination` : Log analytics 活性化の可否 - `true` , `false`

-   `tag` : リソースをカテゴリに分類するためのタグ - 最大 512 文字, `key`:`value`
