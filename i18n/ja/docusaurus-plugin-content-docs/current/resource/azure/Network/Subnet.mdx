## リソース概要

Microsoft Azure では、サブネットは VNet（仮想ネットワーク）の IP アドレス範囲です。 ネットワークトラフィックを分割し、ネットワークリソースを管理できる IP ネットワークの論理的区分です。
サブネットはより大きなネットワーク内のより小さなネットワークであり、仮想ネットワークの異なる部分間でトラフィックを隔離するために使用されます。
Azure の各サブネットには独自の NSG(ネットワークセキュリティグループ)がある可能性があるため、そのサブネットのリソースに対するセキュリティ境界を作ることができます。

## リソース設定値

-   `included_vnet_name` : Subnet が含まれている Virtual Network 名
-   `address_prefixes` : Subnet 使用のための IP アドレス範囲、Virtual Network のアドレス空間内に CIDR 形式で指定された固有のアドレス範囲
-   `private_link_endpoint_enabled` : Subnet に Private Endpoint を接続するかどうかを設定 - `true`, `false`
-   `delegation`
    -   `service_name` : 委任するサービスの名前
    -   `actions` : 委任するアクションのリスト
