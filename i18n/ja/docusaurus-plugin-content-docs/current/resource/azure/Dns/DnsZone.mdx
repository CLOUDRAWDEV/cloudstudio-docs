## リソース概要

Azure DNS 領域はクラウドで DNS ドメインをホストし、DNS レコードを管理できるように Microsoft Azure が提供するサービスです。
A、AAA、CNAME、MX、および TXT のような DNS レコードを管理し、SOA(権限開始)、NS(ネームサーバ)、および PTR(ポインター)レコードのような DNS 設定を構成することができます。
Azure DNS 領域を使用すると、Virtual Machines、Load Balancer、App Services のような他の Azure サービスと統合できる可用性と拡張性に優れた DNS ソリューションを持つことができます。
オンプレミスリソースと一緒に使用して、統合された一貫した DNS 管理環境を提供することもできます。
Azure DNS 領域はパブリックおよびプライベート DNS 領域の両方をサポートします。
パブリック DNS 領域は、ドメイン名をインターネットを通じてアクセスできる IP アドレスにマッピングするために使用されますが、プライベート DNS 領域は、ドメイン名を仮想ネットワーク内でのみアクセスできるプライベート IP アドレスにマッピングするために使用されます。

## リソース設定値

-   `dns_a_record`

    -   `ttl` : DNS Record の Time To Live(秒) - `0` ~ `2147483647`
    -   `records` : IPv4 アドレスのリスト

-   `dns_aaaa_record`

    -   `ttl` : DNS Record の Time To Live(秒) - `0` ~ `2147483647`
    -   `records` : IPv6 アドレスのリスト

-   `dns_cname_record`

    -   `ttl` : DNS Record の Time To Live(秒) - `0` ~ `2147483647`
    -   `record` : サブドメインを別のレコードにリンクするためのエイリアス

-   `dns_ns_record`

    -   `ttl` : DNS Record の Time To Live(秒) - `0` ~ `2147483647`
    -   `records` : NS レコードを構成する値のリスト

-   `dns_ptr_record`

    -   `ttl` : DNS Record の Time To Live(秒) - `0` ~ `2147483647`
    -   `records` : ポインタレコードタイプのドメイン名のリスト

-   `dns_txt_record`

    -   `ttl` : DNS Record の Time To Live(秒) - `0` ~ `2147483647`
    -   `records` : テキストタイプのレコード一リスト

-   `tag` : リソースをカテゴリに分類するためのタグ - 最大 512 文字, `key`:`value`
