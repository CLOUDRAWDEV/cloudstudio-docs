## リソース概要

Azure リポジトリアカウントは、Microsoft Azure が提供するクラウドベースのリポジトリソリューションです。
Azure クラウドで実行されるアプリケーション、仮想マシン、およびその他のサービスのための拡張可能で可用性の高いデータストレージプラットフォームを提供します。
Azure リポジトリアカウントは、Blob リポジトリ、ファイルリポジトリ、キューリポジトリ、テーブルリポジトリなど、様々なタイプのリポジトリサービスをサポートします。
Blob リポジトリは、テキスト、画像、ビデオ、およびその他のバイナリデータなどの非構造化データを保存するために使用されます。

## リソース設定値

-   `account_kind` : Storage Account のタイプ - `StorageV2`,`Storage`

-   `account_tier` : Storage Account 使用のための性能階層 - `Standard`,`Premium`
-   `account_replication_type` : Storage Account のデータ複製タイプ - `LRS`,`GRS`,`RAFRS`,`ZRS`,`GZRS`,`RAGZRS`

-   `access_tier` : アカウントのタイプが BlobStorage/FileStorage/Storage V2 の場合、データ使用パターン指定階層 - `Hot`,`Cool`

-   `min_tls_version` : Storage Account に対してサポートされる最小 TLS バージョン - `TLS1_1`,`TLS1_2`,`TLS1_0`

-   `enable_https_traffic_only` : セキュリティ転送が必要な場合、HTTPS を通じて Azure Storage REST API 作業を呼び出し - `true`,`false`

-   `network_rules`

    -   `default_action` : Network Rules に適用する default action - `Deny`,`Allow`
    -   `bypass` : Logging/Metrics/AzureServices に対してトラフィック迂回の可否 - `AzureServices`,`Logging`,`Metrics`,`None`
    -   `ip_rules` : Network Rules を適用する IP アドレス範囲

-   `storage_container`

    -   `container_access_type` : Storage Container へのアクセスレベル - `private`,`blob`,`container`

-   `storage_queue`

    -   `name` : Queue 名

-   `storage_share`

    -   `quota` : ファイル共有 GB 単位の最大容量 - `1` ~ `5120`

-   `storage_table`

    -   `name` : テーブル名

-- `tag` : リソースをカテゴリに分類するためのタグ - 最大 512 文字, `key`:`value`
