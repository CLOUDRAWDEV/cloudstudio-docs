## リソース概要

Azure Windowsserver Virtual Machine Scale Set は Windows オペレーティングシステムを実行する同一の仮想マシングループであり、 必要に応じて自動的に拡張または縮小することができます。
Scale Set の VM 数を手動で調整するか、リソース使用量（例:CPU、メモリ要求量、またはネットワークトラフィック）に応じて自動的にサイズを調整するルールを定義できます。
これにより、Azure 負荷分散装置で Scale Set の VM インスタンスにトラフィックを分散します。

## リソース設定値

-   `included_vnet_name` : Subnet が含まれている Virtual Network 名
-   `orchestration_mode` : Scale Set によって VM が管理されるかどうかを決定するオーケストレーション方式
-   `platform_fault_domain_count` : 共通電源およびネットワークスイッチを共有する VM のグループ数
-   `vmss_sku` : Virtual Machine Scale Set の価格設定階層
-   `instances` : Scale Set の仮想マシン数 `1`~`1000`
-   `source_image`
    -   `enabled_custom_image` : ユーザーイメージ ID の使用有無 - `true`, `false`
    -   `image_id` : VM 生成のために使用するイメージの ID
    -   `publisher` : VM 生成のために使用するイメージのプロバイダー
    -   `Offer` : VM 生成のために使用するイメージの ID
    -   `sku` : VM 生成のために使用するイメージの価格設定階層
-   `enable_ssh_key_authentication` : 管理者アカウントを認証するために ssh キーを使用
-   `admin_username` : VM に対する管理者ユーザー名
-   `admin_password` : VM に対する管理者パスワード
-   `os_disk_caching` : 内部 OS Disk 使用のための caching タイプ - `ReadWrite`, `ReadOnly`, `None`
-   `storage_account_type` : 内部 OS Disk をバックアップするためのストレージタイプ - `Standard_LRS`, `StandardSSD_LRS`, `Premium_LRS`
-   `disk_size_gb` : 内部 OS Disk のサイズ(GB) - `1` ~ `11400`
-   `network_interface` : Virtual Machine が接続された Network Interface 名
-   `tag` : リソースをカテゴリに分類するためのタグ - 最大 512 文字, `key`:`value`
