## 리소스 개요
Azure Site Recovery는 Microsoft Azure에서 제공하는 재해 복구 솔루션으로, 온프레미스에 있든 다른 클라우드 제공업체에 있든 관계없이 실제 또는 가상 머신에서 실행되는 애플리케이션 및 워크로드를 보호하고 복구하는 데 도움이 됩니다.
Azure Site Recovery를 사용하면 애플리케이션 및 데이터를 보조 Azure 지역과 같은 보조 사이트에 복제하여 정전, 자연 재해 또는 기타 중단 이벤트와 같은 계획되지 않은 가동 중지 시간 동안 장애 조치(failover) 지원을 제공할 수 있습니다.
Azure Site Recovery는 계획된 유지 관리 이벤트에 대한 지원도 제공하므로 가동 중지 시간을 최소화하면서 워크로드를 다른 위치로 마이그레이션할 수 있습니다.




## 리소스 설정 값
- ```sku``` : Recovery Services Vault 사용을 위한 가격 책정 계층 - ```Standard```,  ```RS0```

- ```soft_delete_enabled``` : 데이터 복원을 위한 Soft Delete 활성화 - ```true```,  ```false```

- ```role_assignment```
    + ```scope``` : Role Assignment가 적용되는 리소스 범위
    + ```role_definition_name``` : 할당할 기존 제공 역할 이름


- ```recovery_fabric```
    + ```location``` : Fabic이 위치하는 Azure 영역
 

- ```protection_container```
    + ```recovery_fabric_name``` : Protection Container와 연결된 Fabric 이름
 
- ```replication_policy```
    + ```recovery_point_retention``` : 복구 지점을 보존하는 기간(분)
    + ```snapshot_frequency``` : 애플리케이션 일관성을 위한 앱 일치 스냅샷을 생성하는 주기(분)

- ```container_mapping```
    + ```source_fabric_name``` : 매핑할 Protection Container가 연결된 Source Fabric 이름
    + ```source_container_name``` : 원본 Protection Container 이름
    + ```target_fabric_name``` : 매핑할 Protection Container가 연결된 Target Fabric 이름
    + ```target_container_name``` : 대상 Protection Container 이름
    + ```replication_policy_name``` : 매핑에 사용할 복제 정책 이름
  
- ```network_mapping```
    + ```source_fabric_name``` : 원본 Fabric 이름
    + ```source_vnet_name``` : 원본 가상 네트워크 이름
    + ```target_fabric_name``` : 대상 Fabric 이름
    + ```target_vnet_name``` : 대상 가상 네트워크 이름

- ```public_ip```
    + ```allocation_method``` : 공용 IP 주소 할당 방법 - ```Static```,  ```Dynamic```
    + ```sku``` : 공용 IP의 가격 책정 계층  - ```Basic```,  ```Standard```


- ```replicated_vm```
    + ```source_fabric_name``` : 원본 Fabric 이름
    + ```source_vm_name``` : 복제할 원본 가상머신 이름
    + ```source_container_name``` : 원본 Protection Container 이름
    + ```replication_policy_name``` : 사용할 복제 정책 이름
    + ```target_fabric_name``` : 장애조치가 실행될 때 복제 가상머신이 다뤄지는 대상 Fabric 이름
    + ```target_container_name``` : 장애조치가 실행될 때 가상머신이 복제되는 Protection Container 이름
    + ```storage_account_name``` : 캐싱에 사용될 Storage 계정 이름
    + ```target_disk_type``` : 장애조치가 실행될 때 디스크의 유형
    + ```replica_disk_type``` : 복제 데이터를 저장할 디스크 유형 - ```Standard_LRS```,  ```StandardSSD_LRS```,  ```Premium_LRS```
    + ```source_nic_name``` : 원본 네트워크 인터페이스의 이름
    + ```target_subnet_name``` : 장애조치가 실행될 때 사용되는 대상 Subnet 이름
    + ```target_pip_name``` : 장애조치가 실행될 때 사용하는 공용 IP 이름
