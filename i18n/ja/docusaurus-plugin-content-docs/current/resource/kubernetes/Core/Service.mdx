## リソース概要

Pod への単一の進入点を提供し、それらの Pod 間の通信を管理する抽象化されたクバネティスリソースです。

## リソース設定値

-   `service_name` : Service リソース名
-   `namespace` : Service リソースがデプロイされるネームスペース
-   `annotations` : クバネティスシステムで必要な情報を表示するために使用する key-value
-   `labels` : オブジェクトを区切るためにユーザーが任意に希望する値を指定
-   `selector` : Selector と一致するラベルの key-value を使用してサービストラフィックを Pod にルーティング
-   `type` : サービスのタイプを指定 - `ClusterIP`,`NodePort`,`LoadBalancer`
-   `session_affinity` : クライアントの要求が特定のパッドのみになるように維持されるようにするポリシー - `None`,`clientIP`
-   `port` : サービスによって表示されるポートのリスト
    -   `name` : ポート設定の名前
        -   `protocol` : 該当ポートの IP プロトコル - `TCP`,`UDP`
        -   `port` : サービスによって表示されるポートのリスト
        -   `target_port` : サービスターゲットの Pod からアクセスするポートの番号
