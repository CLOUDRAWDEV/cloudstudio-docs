## リソース概要

特定のストレージ プロビジョナと一緒に動作する動的ストレージ プロビジョニングを定義するクバネティス リソース

### 接続リソース

-   [**PersistentVolume**](https://docs.cstudio.app/ja/docs/resource/kubernetes/Core/PersistentVolume)

## リソース設定値

-   `service_name` : StorageClass リソース名
-   `annotations` : クバネティスシステムで必要な情報を表示するために使用する key-value
-   `labels` : オブジェクトを区切るためにユーザーが任意に希望する値を指定
-   `storage_Provisioner` : ストレージ プロバイダーのタイプ
-   `reclaim_policy` : 使用する回収ポリシー - 指定しないとデフォルトは Delete
-   `volume_binding_mode` : ボリュームバインディングと動的プロビジョニングが発生する必要がある時期 - `Immediate`, `WaitForFirstConsumer`
-   `allow_volume_expansion` : ストレージクラスがボリューム拡張を許可しているかどうかの可否 - `False`, `True`
-   `parameters` : ストレージクラスのボリュームを作成する必要があるプロビジョナのパラメータ
-   `mount_options` : ストレージ クラスによって動的に生成される永続ボリュームには、マウントオプションが指定
