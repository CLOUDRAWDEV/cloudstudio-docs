## 리소스 설정 값

-   `set_container_insights` : Container Insights를 클러스터에 할당할지 설정 - `Enabled`
-   `kms_key_name` : 로컬 클라이언트와 컨테이너 간의 데이터를 암호화하기 위한 AWS Key Management Service 키 이름
-   `logging` : 실행 명령 결과에 대한 로그 리디렉션에 사용할 로그 설정 - `OVERRIDE`,`NONE`,`DEFAULT`
-   `cloud_watch_encryption_enabled` : CloudWatch 로그에서 암호화 활성화 여부 - `false`,`true`
-   `capacity_providers` : 로그를 보낼 CloudWatch 로그 그룹의 이름 - `Fargate`,`Fargate_spot`
-   `service` : 
-   `service.name` : 서비스 이름
-   `service.computing_option` : Service의 작업 분산 방식 - `capacity_provider_stratege`,`launch_type`

-   `service.computing_option.capacity_provider_stratege` 
    -   `service.computing_option.capacity_provider_stratege.capacity_provider` : `Fargate`,`Fargate_spot`   
    -   `service.computing_option.capacity_provider_stratege.base` : 실행할 최소한 작업 수 (100000이하) 
    -   `service.computing_option.capacity_provider_stratege.weight` : 실행된 총 작업 수의 상대적 백분율 (1000 이하)

-   `service.computing_option.launch_type` : Service의 시작 유형
    -   `service.computing_option.launch_type.launch_type` : `ec2`,`Fargate`,`External`   
    -   `service.computing_option.launch_type.task_definition_name` : Service를 사용할 ECS Task Definition의 이름
    -   `service.computing_option.launch_type.desired_count` : 서비스를 실행할 작업 정의의 인스턴스 수
