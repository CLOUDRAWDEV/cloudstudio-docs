## リソース概要

AWS リソースをバックアップする時期と方法を定義するポリシー表現式です。
Backup Plan にリソースを割り当てることができ、AWS バックアップは BackupPlan に従って、そのリソースのバックアップを自動的にバックアップし、維持します。

## リソース設定値

-   `rule.rule_name` : バックアップルール名
-   `rule.backup_vault_name` : バックアップの保存に使用される論理的コンテナ名
-   `rule.frequency_type` : バックアップに関するサイクルのタイミング
-   `rule.frequency_hour` : バックアップ計画を初期化する時間 - `1 以上`
-   `rule.start_time` : バックアップ計画の開始時
-   `rule.start_after_minutes` : バックアップを開始するまでの合計時間 - `60 以上`
-   `rule.complete_after_minutes` : 作業をキャンセルし、エラーを返すまで AWS Backup でバックアップを試行する時間(分) - `121以上`
-   `rule.enabled_use_cold_storage` : cold storage 使用可否
-   `rule.warm_storage_retention_days` : 生成後、リカバリポイントをコールドストレージに移動する指定日数 - `1 ~ 36410`
-   `rule.told_retention_days` : リカバリ地点が作成された後、リカバリ地点を削除する指定日数 - `91 ~ 36500`

-   `resource_assignment.name`
-   `resource_assignment.iam_role_arn` : 対象リソースをリカバリおよびバックアップする時に AWS バックアップで認証するために使用する IAM ロールの ARN
-   `resource_assignment.enabled_contain_all` : すべてのリソースを含めるかどうかを指定 - `false`,`true`
