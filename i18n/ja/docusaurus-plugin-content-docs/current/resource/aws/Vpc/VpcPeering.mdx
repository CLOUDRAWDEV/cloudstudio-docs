## リソース概要

Virtual Private Cloud(VPC)はユーザの AWS アカウント専用の仮想ネットワークです。VPC は AWS クラウドで他の仮想ネットワークと論理的に分離されています。
AWS リソース(例:Amazon EC2 インスタンス)を VPC から開始できます。
VPC ピアリング接続はプライベート IPv4 アドレスまたは IPv6 アドレスを使用して 2 つの VPC 間のトラフィックをルーティングできるようにするための 2 つの VPC 間のネットワーキング接続です。
同一のネットワークに属する場合と同様に、VPC のインスタンスが相互に通信できます。ユーザーの独自 VPC または他の AWS アカウントの VPC と VPC ピアリング接続を作成することができます。
VPC は異なるリージョンにある可能性があります（リージョン間 VPC ピアリング接続とも呼ばれます）。

## リソース設定値

-   `linked_vpc_name_01` : Peering する VPC 名
-   `linked_vpc_name_02` : Peering する VPC 名
-   `auto_accept` : Peering 接続自動承諾の可否부 - `true`, `false`
