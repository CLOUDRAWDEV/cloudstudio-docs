## リソース概要

インターネットゲートウェイは、水平に拡張され、可用性の高い重複 VPC コンポーネントで、VPC とインターネット間の通信を可能にします。 IPv4 トラフィックおよび IPv6 トラフィックをサポートします。
ネットワーク トラフィックに可用性リスクや帯域幅の制約が発生しません。
リソースにパブリック IPv4 アドレスまたは IPv6 アドレスがある場合、インターネットゲートウェイを使用すると、パブリック サブネット（例:EC2 インスタンス）のリソースがインターネットに接続できます。
同様に、インターネットのリソースは、パブリック IPv4 アドレスまたは IPv6 アドレスを使用してサブネットのリソースへの接続を開始できます。
たとえば、インターネットゲートウェイを使用すると、ローカルコンピュータで AWS の EC2 インスタンスに接続できます。
インターネット ゲートウェイは、VPC ルーティング テーブルでインターネット ルーティング可能なトラフィックのターゲットを提供します。
IPv4 通信の場合、インターネットゲートウェイは Network Address Translation（NAT）も行います。
IPv6 を使用する通信の場合、IPv6 アドレスがパブリックであるため、NAT が必要ではありません。
