## リソース概要

AWS VPC(Virtual Private Cloud)の Route Table は VPC 内のサブネットに対するトラフィックをどのような対象に伝達するかを定義する AWS リソースです。
VPC 内のすべてのサブネットは 1 つ以上のルーティングテーブルに接続されており、Routing Table は、Internet Gateway、NAT Gateway、VPC Endpoint、または VPN 接続などの対象に対するルーティング ルールを指定します。
Routing Table はルーティングルールの順にトラフィックを配信し、サブネットで生成されたすべてのトラフィックは、接続されたルーティングテーブルを介してターゲットに配信されます。
これにより、VPC 内のサブネットは安全にインターネットまたは他の VPC と通信できます。

## リソース設定値

-   `included_vpc_name` : Route Table が含まれた VPC の名前
-   `subnet_names` : Route Table に対する説明
-   `linked_route` : Routing パスの名前
    -   `cidr_block` : Routing する Target Resource の CIDR
    -   `internet_gateway_name` : Routing パスを指定する Internet Gateway の名前
    -   `ec2_instance_name` : Routing パスを指定する EC2 Instance の名前
    -   `nat_gateway_name` : Routing パスを指定する NAT Gateway の名前
    -   `vpc_peering_name` : Routing パスを指定する VPC Peering の名前
