## リソース概要

AWS Elasticache Cluster はクラウド基盤のインメモリデータストアであり、迅速かつ拡張可能なキャッシングソリューションを提供します。
Elasticache Cluster は Redis または Memcached エンジンを使用してクラスターモードまたは単一ノードモードで実行できます。
Elasticache Cluster は アプリケーションの性能を向上させるために使用され、データをより速く検索して処理することができます。
Elasticache Cluster は 自動バックアップ、復旧、拡張およびモニタリング機能を提供し、ユーザーが安心してデータを保存できるようにサポートします。

## リソース設定値

-   `engine` : Cache Cluster で使用する Cache Engine の名前 - `redis`, `memcached`
-   `engine_version` : Cache Engine のバージョン
-   `port` : 各 Cache Node と接続するときに使用するポート番号。Memcached は 1121、Redis は 6379 番ポートをデフォルト値として使用
-   `parameter_group_name` : CacheCluster と接続するパラメータグループの名前
-   `node_type` : CacheCluster に使用されたインスタンスクラス
-   `num_cache_nodes` : Cache Cluster の最初の Cache Node の数 - `1`~`40`
-   `az_mode` : Memcached Node Group の Node が単一の可用領域で生成されるのか、それともクラスター地域の様々な可用領域にわたって生成されるのかの可否 - `single-az`, `cross-az`
-   `included_subnet_names` : ElastiCache Cluster をデプロイする Subnet Group に含まれる Subnet の名前
-   `snapshot_retention_limit` : ElastiCache が自動キャッシュクラスタースナップショットを削除する前に保管する期間
