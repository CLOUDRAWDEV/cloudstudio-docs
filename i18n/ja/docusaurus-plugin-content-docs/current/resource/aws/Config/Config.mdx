## リソース概要

AWS アカウントのリソース構成情報を記録し、リソース間の関係を確認するリソース

## リソース設定値

-   `enable_config_recorder` : コンフィギュレーショレコーダーを有効にするか無効にするかの可否 - `false`, `true`
-   `role_name` : AWS Config で使用する Iam Role の名前
-   `recording_method`
-   `recording_method.enable_all_supported` : AWS Config がサポートされているすべての地域リソースタイプに対する構成変更事項を記録しているかどうかの可否 - `false`,`true`
-   `recording_method.exclude_resource_types` : AWS Configuration で構成変更を記録する AWS リソースタイプを指定するリスト
-   `recording_method.include_global_resource_types` : AWS Config が記録するリソースと一緒にサポートされるすべてのタイプのグローバルリソースを含んでいるかどうかの可否 - `false`,`true`
-   `linked_s3_bucket_name` : 構成記録を保存するために使用される S3 バケット名
-   `s3_key_prefix` : 使う S3 バケットの prefix
-   `enable_sns_topic` : AWS Config がアラームを伝達する Sns Topic を有効にするかどうかの可否
-   `delivery_frequency` : AWS Config が構成スナップショットを定期的に送信する頻度
-   `config_rule`
-   `config_rule.name` : config rule の名前
-   `config_rule.rule_owner` : AWS または顧客が AWS 構成ルールを所有·管理しているかどうかの可否 - `AWS`,`customLambda`,`customPolicy`
-   `config_rule.source_identifier` : AWS 構成管理ルールの場合、事前に定義された識別子
