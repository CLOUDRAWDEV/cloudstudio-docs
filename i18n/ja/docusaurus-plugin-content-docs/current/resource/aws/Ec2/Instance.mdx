## リソース概要

Amazon Elastic Compute Cloud(Amazon EC2)は Amazon Web Services(AWS)クラウドで拡張可能なコンピューティング容量を提供します。
Amazon EC2 を使用すると、ハードウェアに先に投資する必要がなく、より迅速にアプリケーションを開発して配布することができます。Amazon EC2 を使用して、必要な数の仮想サーバーを構築し、セキュリティとネットワーキングを構成し、ストレージを管理することができます。
Amazon EC2 では拡張または縮小によって要求事項の変更または使用量のスパイクを処理できるため、トラフィックを予測する必要性が減ります。

## リソース設定値

-   `ami` : インスタンスを実行するために必要な情報を持っている Image(AWS で提供、サポート)
-   `instance_type` : EC2 インスタンスの type
-   `included_subnet_name` : EC2 インスタンスをデプロイする Subnet の名前
-   `security_group_name` : EC2 インスタンスの NIC に指定する Security Group の名前
-   `eip_enable` : EC2 インスタンスに EIP を使用するかどうかの設定 - `true`, `false`
-   `eip_name` : EIP の名前
-   `http_endpoint` : メタデータサービスを使用できるかどうかの設定 - `enabled`, `disabled`
-   `http_tokens` : メタデータサービスにセッショントークンが必要かどうかを設定 - `required`, `optional`
-   `root_volume_type` : Root Volume のタイプ - `gp2`, `gp3`, `io1`, `io2`, `sc1`, `st1`, `standard`
-   `root_volume_size` : Root Volume の Size (GiB)
-   `ebs_block_device`
    -   `volume_size` : EBS Volume の Size (GiB)
    -   `volume_type` : EBS Volume のタイプ
