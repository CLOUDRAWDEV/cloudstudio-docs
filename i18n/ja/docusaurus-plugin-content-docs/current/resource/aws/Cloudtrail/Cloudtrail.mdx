## リソース概要

AWS CloudTrail を使用すると AWS Management Console、AWS SDK、コマンドラインツールおよび上位レベルの AWS サービスを通じて
実行された API 呼び出しを含め、アカウントに対する AWS API 呼び出し記録を取得することで、クラウドで AWS 配布状況をモニタリングすることができます。
また、どのユーザやアカウントが CloudTrail をサポートするサービスに対して AWS API を呼び出したのか、どのソース IP アドレスに呼び出しが行われたのか、
いつ呼び出しが発生した時間を確認することができます。 API を使用して CloudTrail をアプリケーションに統合し、組織に対する追跡生成を自動化し、
追跡状態を確認し、管理者が CloudTrail ロギングを有効にし、無効にする方法を制御することができます。

## リソース設定値

-   `linked_s3_bucket_name` : ログファイルの publish 用に指定された S3 バケットの名前
-   `s3_key_prefix` : ログファイル転送のために指定したバケット名の後に来る S3 キー接頭辞
-   `multi_region_enabled` : multi-region 追跡活性化
-   `global_service_events` : Cloudtrail が IAM のようなグローバルサービスのイベントをログファイルに掲示するかどうかの可否 - `true`, `false`
