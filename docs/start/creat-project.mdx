# 새 프로젝트 생성하기
<hr />

클라우드 스튜디오에서는 하나의 캔버스를 📄프로젝트라고 부릅니다. 

프로젝트를 생성하려면 프로젝트를 관리할 수 있는 📁폴더가 먼저 생성되어 있어야 합니다.

폴더를 생성하는 방법은 [여기](#create-folder)를 참고해 주세요.

## 프로젝트를 생성하는 세 가지 방법

클라우드 스튜디오 프로젝트(캔버스)를 생성하는 방법은 아래의 세 가지가 있습니다.
- [제공하는 템플릿으로 생성하기](#create-template)
- [빈 프로젝트 생성하기](#creat-empty)
- [기존 인프라 가져오기(import)](#creat-import)


**탬플릿을 사용**하면 Landing Zone, MSA, Load Balancer, Ubuntu Virtual Machine, CDN 등 원하는 카테고리의 클라우드 인프라를 빠르게 생성할 수 있습니다. 템플릿에 대한 자세한 설명은 여기(링크)를 참고해 주세요.

**템플릿 없이 직접 캔버스를 구성**하고 싶다면 빈 프로젝트 생성하기를 사용합니다. 빈 캔버스에 원하는 리소스를 드래그 앤 드롭하여 클라우드 인프라를 구성합니다.

**기존 인프라를 캔버스로 가져오고 싶다면** 캔버스 페이지에 접속한 후 크레덴셜을 선택하여 가져옵니다.


### 제공하는 템플릿으로 생성하기 {#create-template}

1.  먼저 **프로젝트 생성하기 버튼**을 클릭하여 프로젝트 생성 팝업을 불러옵니다.



1.  프로젝트명을 작성합니다. 

> 프로젝트명은 다음 조건을 만족해야 합니다. 
✅  영문, 한글, 숫자만 허용
✅  공백 없이
✅  2 ~ 20 자리
> 
1. 프로젝트를 담을 폴더를 선택합니다.

1.  프로젝트에 대한 설명을 입력합니다. 최대 100글자 까지 입력 가능합니다.


1.  CSP와 카테고리를 선택합니다.


> CSP란?
Cloud Service Provider의 약자로, **공공 클라우드 인프라, 플랫폼 서비스를 제공하는 업체**를 의미합니다. 클라우드 스튜디오에서 제공하는 CSP는 Micosoft의 Azure, Naver의 NAVER CLOUD PLATFORM, Amazon의 Amazon Web Services 입니다.
> 

> 제공하는 템플릿에 대한 자세한 설명은 여기(링크)를 참고해 주세요.
> 

> 💡 템플릿 정렬을 바꿔서 볼 수 있습니다. 다음 버튼들을 클릭해보세요!
> 

1.  프로젝트 생성하기 버튼을 클릭합니다.


### 빈 프로젝트 생성하기 {#create-empty}

1.  먼저 **프로젝트 생성하기 버튼**을 클릭하여 프로젝트 생성 팝업을 불러옵니다.


1.  프로젝트명을 작성합니다. 

> 프로젝트명은 다음 조건을 만족해야합니다. 
✅  영문, 한글, 숫자만 허용
✅  공백 없이
✅  2 ~ 20 자리
> 
1.  프로젝트를 담을 폴더를 선택합니다.

1.  프로젝트에 대한 설명을 입력합니다. 최대 100글자 까지 입력 가능합니다.

1.  프로젝트 생성하기 버튼을 클릭합니다.

### 기존 인프라 가져오기(import){#create-import}
기존 인프라를 가져오기 위해선 캔버스에 접속해야 합니다. 먼저 빈 프로젝트를 생성합니다. 빈 프로젝트를 생성하는 방법은 를 참고해주세요.

인프라를 가져오는 방법에는 두 가지가 있습니다. 파일에서 불러오기는 현재 서비스 준비중입니다.

- 인프라에서 불러오기
- 파일에서 불러오기

**인프라에서 불러오기**

1.  빈 프로젝트를 클릭하여 캔버스 페이지에 접속합니다.
2.  크레덴셜을 생성합니다. 기존 인프라를 가져오려면 크레덴셜이 필수적으로 생성되어 있어야 합니다. 크레덴셜 생성 방법은를 참고해주세요.
3.  파일 > 가져오기를 클릭합니다.


1.  팝업창이 뜨면 크레덴셜을 선택합니다.


1.  리소스그룹 조회 버튼을 클릭한 후 리소스그룹 목록에서 원하는 리소스그룹을 선택합니다.


1.  가져오기 버튼을 클릭하여 기존 인프라 가져오기를 완료합니다.

파일**에서 불러오기**

파일에서 불러오기 기능은 서비스 준비중입니다.
