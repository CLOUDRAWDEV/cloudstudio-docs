## 크레덴셜 생성하기 {#create-credential}

CSP ```크레덴셜 생성```은 아래의 🔗 링크를 참고해 주세요!

- [Azure 크레덴셜 생성하기](https://docs.microsoft.com/en-us/azure/automation/shared-resources/credentials?tabs=azure-powershell#create-a-new-credential-asset)

- [NCloud 크레덴셜 생성하기](https://api.ncloud-docs.com/docs/common-ncpapi)

## 크레덴셜 등록하기
크레덴셜 등록은 ```메인 페이지```에서 할 수도 있고 ```캔버스 페이지```에서 할 수도 있습니다.
크레덴셜을 미리 등록해두는 이유는 이후 캔버스를 배포할 때와 기존 인프라를 임포트 해올 때 크레덴셜 정보가 필요하기 때문입니다.


### Azure 크레덴셜 등록하기
각각의 필드 옆에 있는 🔗 링크 아이콘을 클릭하면 ```azure docs```로 연결됩니다.
참고하여 필드를 채워줍니다. 아직 Azure 크레덴셜을 생성하지 않았다면 [여기](#create-credential)를 참고하세요.

<div className={styles.imgWrap}>
    <img src={ registration01 } alt="이미지" />
</div>

<br />

1. 클라우드 스튜디오에서 ```크레덴셜 등록``` 버튼을 클릭합니다.
2. azure를 선택합니다.
3. 크레덴셜 이름을 입력합니다.

> - 영문으로 시작해야 합니다.
> - 숫자/언더바/하이픈 외 특수기호 사용은 불가합니다.
> - 4 ~ 20 자리로 입력합니다.

4. ```Azure Potal```에 접속한 후 로그인 합니다. https://azure.microsoft.com/ko-kr/get-started/azure-portal/
5. ```Azuer Active Directory```를 클릭합니다.

<div className={styles.imgWrap}>
    <img src={ registration02 } alt="이미지" />
</div>

<br />

6. ```앱등록```을 클릭한 후 ```디렉터리(테넌트) ID```를 클라우드 스튜디오 ```Tenant ID```에 입력합니다. ```애플리케이션(클라이언트) 아이디```를 클라우드 스튜디오 ```Service Principal ID```에 입력합니다.

<div className={styles.imgWrap}>
    <img src={ registration03 } alt="이미지" />
</div>

<br />

7. 왼쪽 사이드 바에서 ```인증서 및 암호```를 클릭합니다.
8. ```새 클라이언트 암호```를 클릭합니다.
9. Key에 대한 간단한 설명을 입력한 후 만료시간을 선택합니다. 입력이 완료되면 ```추가```를 클릭합니다.
10.  생성된 값을 복사하여 클라우드 스튜디오 Service Principal Key에 붙여넣기 합니다.

<br />

:::caution

생성된 Service Principal Key는 페이지를 ```새로고침하면 볼 수 없습니다```. 바로 적어두는 것을 권장합니다. Key 값이 기억나지 않는다면 다시 생성할 수 있지만 권장사항은 아닙니다.

:::

<br />

11. Azure Potal ```홈```으로 돌아와 ```구독```을 클릭합니다.

<div className={styles.imgWrap}>
    <img src={ registration04 } alt="이미지" />
</div>

<br />

12. ```구독 ID```를 클라우드 스튜디오 ```Subscription ID```에 입력합니다. 

<div className={styles.imgWrap}>
    <img src={ registration05 } alt="이미지" />
</div>

<br />

13. ```등록하기```를 클릭합니다.
