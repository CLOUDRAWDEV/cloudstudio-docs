크레덴셜 생성 및 등록하기​

클라우드 스튜디오에 크레덴셜을 등록하기 위해서는 각 CSP 페이지에서 크레덴셜을 생성해야 합니다. 크레덴셜 등록은 메인 페이지에서 할 수도 있고 캔버스 페이지에서 할 수도 있습니다. 크레덴셜을 미리 등록해두는 이유는 이후 캔버스를 배포할 때와 기존 인프라를 임포트 해올 때 크레덴셜 정보가 필요하기 때문입니다.


크레덴셜이란?
사용자 로그인 정보 등 개인 신상을 암호화한 정보입니다.

 

Azure 크레덴셜 등록하기​

각각의 필드 옆에 있는 🔗 링크 아이콘을 클릭하면 azure docs로 연결됩니다. 참고하여 필드를 채워줍니다. 아직 Azure 크레덴셜을 생성하지 않았다면 여기를 참고하세요.


  1. 클라우드 스튜디오에서 크레덴셜 등록 버튼을 클릭합니다.
  2. azure를 선택합니다.
  3. 크레덴셜 이름을 입력합니다.
  • 영문으로 시작해야 합니다.
  • 숫자/언더바/하이픈 외 특수기호 사용은 불가합니다.
  • 4 ~ 20 자리로 입력합니다.
  1. Azure Potal에 접속한 후 로그인 합니다. https://azure.microsoft.com/ko-kr/get-started/azure-portal/
  2. Azuer Active Directory를 클릭합니다.

  1. 앱등록을 클릭한 후 디렉터리(테넌트) ID를 클라우드 스튜디오 Tenant ID에 입력합니다. 애플리케이션(클라이언트) 아이디를 클라우드 스튜디오 Service Principal ID에 입력합니다.

  1. 왼쪽 사이드 바에서 인증서 및 암호를 클릭합니다.
  2. 새 클라이언트 암호를 클릭합니다.
  3. Key에 대한 간단한 설명을 입력한 후 만료시간을 선택합니다. 입력이 완료되면 추가를 클릭합니다.
  4. 생성된 값을 복사하여 클라우드 스튜디오 Service Principal Key에 붙여넣기 합니다.

caution

생성된 Service Principal Key는 페이지를 새로고침하면 볼 수 없습니다. 바로 적어두는 것을 권장합니다. Key 값이 기억나지 않는다면 다시 생성할 수 있지만 권장사항은 아닙니다.


  1. Azure Potal 홈으로 돌아와 구독을 클릭합니다.

  1. 구독 ID를 클라우드 스튜디오 Subscription ID에 입력합니다.

  1. 등록하기를 클릭합니다.
tip

Azure CLI로 크레덴셜을 등록하는 방법은 아래 링크를 참고해주세요. https://docs.microsoft.com/ko-kr/cli/azure/what-is-azure-cli

 

NCloud 크레덴셜 등록하기​

클라우드 스튜디오를 사용해 NAVER Cloud Platform로 배포하기 위해서는 NAVER Cloud Platform의 API 인증키인 Access key와 Secret Key가 필요합니다. API 인증키는 API를 호출한 사용자가 권한을 가진 사용자인지 식별하는 도구입니다.


  1. 크레덴셜명을 입력합니다.
  2. 인증키 생성을 위해 NAVER Cloud Platform 포털에서 회원가입을 진행합니다.
tip

NAVER Cloud Platform 회원가입 방법은 여기를 참고해주세요.


  1. NCP사이트에서 이용 신청하기를 클릭합니다. 단, 결제수단을 필수로 등록해야 인증키를 발급받을 수 있습니다.

  1. 포털 > 마이페이지 > 계정관리 > 인증키 관리 페이지에 접속합니다.
  2. 인증키가 없을 경우 신규 API 인증키 생성 버튼을 클릭합니다.

  1. Access Key ID를 복사하여 클라우드 스튜디오에 붙여넣기 합니다.
  2. Secret Key를 생성하여 클라우드 스튜디오에 붙여넣기 합니다.

tip

모든 권한을 갖고 있는 메인 계정이 아닌 서브 계정에서 API 인증키를 발급하고, 인증키 유출에 대비하여 주기적으로 교체하는 것을 권장합니다. API 인증키를 사용하지 않거나, 도용이 의심되는 경우에도 보안을 위해 발급한 인증키를 중지하거나 삭제하는 것이 좋습니다. API 인증키를 새롭게 생성한 후에도 사용 중이던 서비스를 정상적으로 이용하려면 변경된 API 인증키를 적용해야 합니다.

 

AWS 크레덴셜 등록하기​

  1. https://console.aws.amazon.com/iam/ 에서 IAM에 로그인하여 콘솔을 엽니다.

  1. 콘솔 홈에서 우측 상단에 내계정을 클릭한 후 보안 자격 증명을 선택합니다.

  1. 액세스 키를 클릭합니다.
  2. 새 액세스 키 만들기를 클릭합니다.
  1. 액세스 키 표시를 클릭하거나 엑셀 파일을 다운 받아 key를 확인합니다.

caution

새 액세스 키 ID와 보안 액세스 키가 포함된 키 파일을 처음 키 생성 시 다운로드하지 않으면 보안 액세스 키를 다시 가져올 수 없습니다.
