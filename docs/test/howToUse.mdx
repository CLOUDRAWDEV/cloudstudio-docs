# How To Use
<hr/>
Docs에 관한 내용은 docs폴더에 작성합니다. 사이드바에 메뉴를 추가하고 싶다면 폴더안 아무 mdx파일을 카피합니다. md는 단순히 텍스트를 쓸 수 있는 파일이고 mdx는 Markdown 파일 내에서 JSX를 작성하고 이를 React 구성 요소로 렌더링할 수 있습니다.

**sidebar_position**은 사이드바에 나타날 **메뉴 순서**입니다. 지금은 sidebar_position이 1로 설정되어 있으니 사이드바에서 가장 상단에 나타나고 있습니다.



## 마크다운 사용법 
```jsx title="터미널에서"
npm run start
```

Notion에서 글을 쓴 후 md로 파일을 내보내서 글을 복붙시킵니다. 이미지 경로는 다시 작성해줘야 합니다.

### 제목과 목차
텍스트 사이즈는 아래와 같이 샵의 개수에 따라 크기를 변경할 수 있습니다.

```js
## 제목 2

### 제목 3

#### 제목 4

본문은 샵없이 작성합니다.
```


각 제목에는 자동으로 생성되거나 명시적으로 지정될 수 있는 ID가 있습니다.
표제 ID를 사용하면 Markdown 또는 JSX의 특정 문서 표제에 연결할 수 있습니다.
특별한 Markdown 구문을 사용하면 명시적 제목 ID 를 설정할 수 있습니다 .

```jsx title="명시적 아이디 생성하기"
### Hello World {#my-explicit-id}
```

```jsx title="특정 문서 표제에 링크걸기"
[link](#heading-id)
<Link to="#heading-id">link</Link>
```

인라인 목차 표시하기. 페이지 오른쪽에 현재 페이지 목차를 나타내주고 있지만, 컴포넌트를 사용하여 페이지 내에 목차를 써줄 수 있습니다.
```js
import TOCInline from '@theme/TOCInline';

<TOCInline toc={toc} />
```

### 코드블록 
코드 블록은 3개의 백틱 문자열로 둘러싸인 텍스트 블록입니다. title 사이에 공백을 둡니다.

**제목이 있는 경우**
```jsx title="/src/components/HelloCodeTitle.js"
function HelloCodeTitle(props) {
  return <h1>Hello, {props.name}</h1>;
}
```

**제목이 없는 경우**
```js
console.log('Every repo must come with a mascot.');
```

### 이미지 삽입
이미지 파일은 ```src > static > img```에서 관리합니다.
docs에 img 폴더를 만들어 메뉴별로 관리할 수도 있습니다. ```예) docs > start > img 폴더를 생성```
먼저 컴포넌트 경로를 불러온 후 컴포넌트를 작성합니다. 이미지 동시에 불러오는 방법 아직은 없음.

<div className={styles.imgWrap}>
    <img width="450px" height="300px" src={ edit } alt="이미지" />
</div>

**scss도 같이 import해와야 함**
