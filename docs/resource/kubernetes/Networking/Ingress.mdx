## 리소스 개요

인바운드 연결이 백엔드에서 정의한 엔드포인트에 도달하도록 허용하는 규칙에 관한 쿠버네티스 리소스

## 연관 리소스

-   [**Cluster**](https://docs.cstudio.app/docs/resource/kubernetes/Base/Cluster)
-   [**Namespace**](https://docs.cstudio.app/docs/resource/kubernetes/Core/Namespace)


### 연결 리소스

-   [**IngressClass**]
-   [**NginxIngress**](https://docs.cstudio.app/docs/resource/kubernetes/Helm/NginxIngress)


## 리소스 설정 값

-   `service_name` : Ingress 리소스 이름
-   `namespace` : Ingress 리소스가 배포될 네임스페이스
-   `annotations` : 쿠버네티스 시스템에서 필요한 정보들을 표시해 주기 위해서 사용하는 key-value
-   `labels` : 객체를 구분하기 위해서 사용자가 임의로 원하는 값을 지정
-   `tls` : TLS를 적용
    -   `hosts` : TLS 인증서에 포함된 호스트 목록
    -   `secret_name` : TLS 인증서와 관련된 쿠버네티스 시크릿 이름 지정
-   `ingress_class_name` : 연결할 인그레스 클래스 이름을 지정
-   `host` : 라우팅 규칙을 적용할 호스트네임 주소를 지정
-   `backend` : 라우팅할 백엔드 정보를 입력
    -   `name` : 라우팅 rule 이름
        -   `path` : 백엔드와 연결될 경로
        -   `path_type` : 경로 일치의 해석을 결정 - `Prefix`, `ImplementationSpecific`, `Exact`
        -   `port` : 서비스의 포트 번호 - `1~65535`
