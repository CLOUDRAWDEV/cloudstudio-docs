## 리소스 개요

Amazon Elastic Kubernetes Service(Amazon EKS)는 Kubernetes 클러스터를 설치하고 운영할 필요 없이 AWS에서 Kubernetes를 손쉽게 실행할 수 있는 관리형 서비스입니다.

## 리소스 설정 값

-   `included_subnet_names` : EKS Cluster를 배포할 Subnet Group에 포함될 Subnet의 이름
-   `enabled_cluster_log_types` : 활성화할 Control Plane 로깅 목록 - `api`, `audit`, `authenticator`, `controllerManager`, `scheduler`
-   `encryption_key_name` : Cluster에서 사용할 KMS key의 이름
-   `endpoint_public_access` : EKS public API 서버 엔드포인트 활성화 여부 설정 - `true`, `false`
-   `public_access_cidrs` : EKS public API 서버 엔드포인트에 액세스할 수 있는 CIDR 블록을 지정
-   `node_group` : Cluster Node Group 이름, 영문, 숫자, 하이픈(-), 밑줄(\_) 사용 가능
    -   `desired_size` : 원하는 EKS 작업자 노드 수
    -   `max_size` : 최대 EKS 작업자 노드 수
    -   `min_size` : 최소 EKS 작업자 노드 수
    -   `max_unavailable` : 노드 그룹 업데이트 중 사용할 수 없는 작업자 노드의 원하는 최대 수
-   `fargate_profile` : EKS Fargate 프로필의 이름, 영문, 숫자, 하이픈(-), 밑줄(\_) 사용 가능
    -   `namespace` : Selector에서 사용할 Kubernetes Namespace의 이름
