## 리소스 개요

Amazon DynamoDB는 완벽하게 관리되는 NoSQL 데이터베이스 서비스로, 원활한 확장성과 함께 빠르고 예측 가능한 성능을 제공합니다. Amazon DynamoDB를 사용하여 원하는 양의 데이터를 저장하거나 가져올 수 있는 데이터베이스 테이블을 만들고, 어떤 수준의 요청 트래픽도 처리할 수 있습니다. Amazon DynamoDB는 테이블의 데이터와 트래픽을 충분한 수의 서버로 자동 분산하여 고객이 지정한 요청 용량과 저장된 데이터 양을 처리하면서도 일관되고 빠른 성능을 유지합니다.

## 리소스 설정 값

-   `billing_mode` : DynamoDB의 읽기 및 쓰기 처리량에 대한 요금 부과 방식과 용량 관리 방식 - `PROVISIONED`,`PAY_PER_REQUEST`
-   `read_capacity` : DynamoDB Table의 읽기 단위 수
-   `write_capacity` : DynamoDB Table의 쓰기 단위 수
-   `hash_key` : 해시(파티션) 키로 사용할 속성
-   `hash_key_type` : 해시(파티션) 키로 사용할 속성 - `S`, `N`, `B`
-   `range_key` :범위(정렬) 키로 사용할 속성
-   `range_key_type` : 해시(파티션) 키로 사용할 속성 - `S`, `N`, `B`
-   `point_in_time_recovery_enabled` : 특정 시점 복구 옵션 활성화 - `true`, `false`
-   `server_side_encryption` : Server Side Encryption 이름
    -   `encryption_enabled` : AWS 관리형 KMS 고객 마스터 키를 사용하여 저장 데이터 암호화 활성화 여부 - `true`, `false`
    -   `kms_key_name` : AWS KMS 암호화에 사용해야 하는 CMK의 이름
