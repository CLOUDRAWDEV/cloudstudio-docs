## 리소스 개요

Amazon API Gateway를 사용하면 모든 규모에서 자체 REST 및 WebSocket API를 생성하고 배포할 수 있습니다. AWS 클라우드에 저장된 데이터뿐만 아니라 Amazon Web Services 또는 기타 웹 서비스에 액세스하는 강력하고 안전하며 확장 가능한 API를 생성할 수 있습니다. 자체 클라이언트 애플리케이션에서 사용할 API를 생성하거나 타사 앱 개발자가 API를 사용할 수 있도록 할 수 있습니다.

## 리소스 설정 값

-   `protocol_type` : API Gateway의 Protocol- `HTTP`,`WEBSOCKET`
-   `route_selection_expression` : API에 대한 경로 선택 표현식
-   `integration` : API Gateway Integration의 이름
    -   `integration_type` : API Gateway Integration의 유형 - `HTTP_PROXY`, `HTTP`, `AWS`, `AWS_PROXY`, `MOCK`
    -   `integration_method` : API Gateway Integration의 HTTP Method
    -   `integration_uri` : Lambda 프록시 통합을 위한 Lambda 함수의 URI, HTTP 통합은 정규화된 URL 지정, HTTP API 프라이빗 통합, ELB의 ARN 지정
-   `route` : API Gateway Route의 이름
    -   `route_key` : Route Key, HTTP API의 경우 경로 키는 $default이거나 HTTP 메서드와 리소스 경로(GET /pets)의 조합을 사용
    -   `included_integration_name` : Route를 연결할 API Gateway Integration 이름
-   `stage` : API Gateway Stage 이름, 1 ~ 128자 사이
