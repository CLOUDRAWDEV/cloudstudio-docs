## 리소스 개요

AWS의 IAM (Identity and Access Management) 리소스는 보안을 강화하기 위해 다양한 비밀번호 정책을 설정할 수 있습니다. 이를 통해 AWS 계정에 대한 액세스를 보호하고 인증된 사용자만이 계정에 액세스할 수 있도록 보장합니다. IAM 계정 비밀번호 정책은 비밀번호의 최소 길이, 복잡성, 만료 기간 등을 정의할 수 있습니다. 이 정책은 AWS Console, AWS CLI, API 등을 통해 적용할 수 있으며, 적용된 비밀번호 정책은 모든 IAM 사용자에게 적용됩니다.

## 리소스 설정 값

-   `minimum_password_length` : Password 최소 길이
-   `require_numbers` : Password에 숫자가 반드시 포함할지 설정 - `true`, `false`
-   `require_symbols` : Password에 특수 문자가 반드시 포함할지 설정 - `true`, `false`
-   `require_lowercase_characters` : Password에 소문자가 반드시 포함할지 설정 - `true`, `false`
-   `require_uppercase_characters` : Password에 대문자가 반드시 포함할지 설정 - `true`, `false`
-   `allow_users_to_change_password` : 사용자가 자신의 비밀번호를 변경할 수 있도록 허용할지 설정 - `true`, `false`
-   `password_reuse_prevention` : 사용자가 재사용을 금지한 이전 비밀번호의 수
-   `max_password_age` : 사용자 비밀번호가 유효한 일수
