## 리소스 개요

Cloudfront Cache Policy는 Cloudfront 배포의 캐시 동작에 연결하여 CloudFront의 사용자 지정을 설정하는 캐시 정책입니다.

## 연관 리소스

### 상위 리소스

-   [**Base**](https://docs.cstudio.app/docs/resource/aws/Base)

### 연결 리소스

-   [**CloudfrontDistribution**](https://docs.cstudio.app/docs/resource/aws/Cloudfront/CloudfrontDistribution)

## 리소스 설정 값

-   `min_ttl` :객체가 CloudFront 캐시에 남아 있어야 하는 최소 시간(초)
-   `max_ttl` : 객체가 CloudFront 캐시에 유지되는 최대 시간(초)
-   `default_ttl` : 객체가 CloudFront 캐시에 남아 있어야 하는 기본 시간(초)
-   `cookie_behavior` : 쿠키를 오리진 서버로 전달할지 여부 - `whitelist`,`all`,`none`,`allExcept`
-   `header_behavior` : 지정된 헤더의 값을 기반으로 객체를 캐싱할지 여부 - `whitelist`,`none`
-   `query_string_behavior` : 쿼리 문자열 파라미터 값을 기준으로 콘텐츠의 다른 버전을 캐싱할지 여부 - `whitelist`,`all`,`none`,`allExcept`
