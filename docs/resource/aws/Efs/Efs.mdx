## 리소스 개요

파일 시스템을 생성, EC2 인스턴스에 파일 시스템을 탑재, EC2 인스턴스에서 데이터를 읽고 파일 시스템에 작성하거나 파일을 공유할 수 있습니다.

## 리소스 설정 값

-   `create_token` : 파일 시스템 작성을 보장하기 위해 Elastic 파일 시스템을 작성할 때 참조로 사용되는 고유한 이름
-   `availability_zone_name` : 파일 시스템을 생성할 AWS 가용성 영역
-   `enabled_encrypted` : 파일 시스템 암호화 여부 - `false`,`true`
-   `throughput_mode` : 파일 시스템의 처리량 모드 - `provisioned`,`elastic`, `bursting`
-   `provisioned_throughput_in_mibps` : 파일 시스템에 프로비저닝할 처리량(MiB/s 단위) - `1-3414`
-   `performance_mode` : 파일 시스템 성능 모드 - `generalPurpose`, `maxIO`
-   `lifecycle_policy` : 수명 주기 관리, 액세스 패턴의 변화에 따라 자동으로 비용을 절감
    -   `transition_to_ia` : 파일을 IA 스토리지 클래스로 전환하는 데 걸리는 시간 - `1`,`7`,`14`,`30`,`60`
    -   `ransition_to_primary_storage_class` : 파일을 액세스 저장소에서 기본 저장소로 전환하는 데 사용되는 정책 - `None`,`FirstAccess`
-   `mount_target_subnets` : 마운트 대상을 추가할 서브넷 목록
-   `file_system_policy` : EFS 파일 시스템에 대한 JSON 포맷 파일 시스템 정책
-   `enable_backup_policy` : 백업 정책 사용 여부 - `false`,`true`
-   `enable_replica` : 복제 생성 여부 - `false`,`true`
-   `access_point` : 공유 데이터 세트에 대한 애플리케이션 액세스를 보다 쉽게 관리할 수 있도록 하는 EFS 파일 시스템의 애플리케이션별 진입점
    -   `root_directory_path` : 액세스 포인트를 사용하여 EFS 파일 시스템에 액세스하는 NFS 클라이언트에 루트 디렉터리로 노출할 EFS 파일 시스템의 경로
    -   `posix_user_uid` : 이 액세스 포인트를 사용하는 모든 파일 시스템 작업에 사용되는 POSIX 사용자 ID - `4294967295이하`
    -   `posix_user_gid` : 이 액세스 포인트를 사용하는 모든 파일 시스템 작업에 사용되는 POSIX 그룹 ID - `4294967295이하`
    -   `posix_user_secondary_gids` : 이 액세스 포인트를 사용하는 모든 파일 시스템 작업에 사용되는 보조 POSIX 그룹 ID
    -   `root_owner_uid` : Root Directory에 적용할 POSIX 사용자 ID - `4294967295이하`
    -   `root_owner_gid` : Root Directory에 적용할 POSIX 그룹 ID - `4294967295이하`
    -   `root_permissions` : 파일의 모드 비트를 나타내는 8진수 형식의 Root Directory에 적용할 수 있는 POSIX 권한
