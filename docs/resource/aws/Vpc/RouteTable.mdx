## 리소스 개요

AWS VPC(Virtual Private Cloud)의 Route Table은 VPC 내의 서브넷에 대한 트래픽을 어떤 대상으로 전달할지 정의하는 AWS 리소스입니다. VPC 내의 모든 서브넷은 하나 이상의 라우팅 테이블에 연결되어 있으며, 라우팅 테이블은 인터넷 게이트웨이, NAT 게이트웨이, VPC 엔드포인트 또는 VPN 연결과 같은 대상에 대한 라우팅 규칙을 지정합니다. 라우팅 테이블은 라우팅 규칙의 순서대로 트래픽을 전달하며, 서브넷에서 생성된 모든 트래픽은 연결된 라우팅 테이블을 통해 대상으로 전달됩니다. 이를 통해 VPC 내의 서브넷들은 안전하게 인터넷 또는 다른 VPC와 통신할 수 있습니다.

## 리소스 설정 값

-   `included_vpc_name` : Route Table이 포함된 VPC의 이름
-   `subnet_names` : Route Table에 대한 설명
-   `linked_route` : Routeing 경로의 이름
    -   `cidr_block` : Routeing할 Target Resource의 CIDR
    -   `internet_gateway_name` : Routeing 경로를 지정할 Internet Gateway 이름
    -   `ec2_instance_name` : Routeing 경로를 지정할 EC2 Instance 이름
    -   `nat_gateway_name` : Routeing 경로를 지정할 NAT Gateway 이름
    -   `vpc_peering_name` : Routeing 경로를 지정할 VPC Peering 이름
