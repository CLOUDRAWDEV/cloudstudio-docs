## 리소스 개요
Azure Database for MySQL - 유연한 서버는 데이터베이스 관리 기능 및 구성 설정에 대한 보다 세부적인 컨트롤과 유연성을 제공하도록 설계된 완전 관리형 프로덕션-레디 데이터베이스 서비스입니다. 
유연한 서버는 다음에 가장 적합합니다.
  + 백업, 고가용성, 보안 및 모니터링과 같은 기능에 대한 배포 용이성, 기본 크기 조정 및 낮은 데이터베이스 관리 오버헤드
  + 더 나은 컨트롤 및 사용자 지정을 통해 커뮤니티 버전의 MySQL이 필요한 애플리케이션 개발
  + 동일한 영역, 영역 중복 고가용성 및 관리되는 유지 관리 기간의 프로덕션 워크로드
  + 간소화된 개발 환경
  + Enterprise급 보안, 규정 준수 및 개인 정보

## 리소스 설정 값
- ```version``` : 사용할 MySQL 버전 - ```5.7```, ```8.0.21``

- ```sku_name``` : MySQL 가격 책정 계층 이름, Tier + 컴퓨팅 세대 + 코어 수의 조합 - ```azureMysqlFlexibleSku```

- ```storage_mb``` : MySQL Server의 용량(MB) - ```storageGB```

- ```iops``` : MySQL Flexible Server의 스토리지 IOPS - ```360``` ~ ```20000```

- ```admin_username``` : Creation mode가 default인 경우 Server의 관리자 사용자 이름

- ```admin_password``` : Creation mode가 default인 경우 Server의 관리자 로그인 암호

- ```auto_grow_enabled``` : 스토리지 자동 증가 활성화 - ```true``` , ```false```

- ```availability_zone``` : 가용 영역을 지정합니다. 가능한 값은 1, 2, 3입니다. - ```1``` ~ ```3```

- ```enable_high_availability``` : MySQL Flexible Server의 고가용성 활성화 여부  - ```true``` , ```false```

- ```high_availability_mode``` : MySQL Flexible Server의 고가용성 모드  - ```SameZone``` , ```ZoneRedundant```

- ```standby_availability_zone``` : 대기 서버의 가용 영역을 지정합니다. 가능한 값은 1, 2, 3입니다. 가용 영역과 중복될 수 없습니다.  - ```1``` ~ ```3```

- ```backup_retention_days``` : MySQL Flexible Server의 백업 보존 일수입니다. 가능한 값은 1~35일입니다.  - ```1``` ~ ```35```

- ```backup_retention_days``` : MySQL Flexible Server의 백업 보존 일수입니다. 가능한 값은 1~35일입니다.  - ```1``` ~ ```35```

- ```public_network_access_enabled``` : Server에 퍼블릭 네트워크 접근 허용 - ```true``` , ```false```

- ```delegated_subnet_name``` : 가상 네트워크 위임을 위한 Subnet 이름

- ```private_dns_zone_name``` : MySQL Flexible Server를 생성하기 위한 프라이빗 DNS 영역

- ```configuration``` 
  + ```name``` : MySQL 유연한 서버 구성의 이름
  + ```param_value``` : MySQL 유연한 서버 구성의 값


- ```database```
  + ```charset``` : MySQL Database의 문자 집합 - ```azureMysqlChartSet``` 
  + ```collation``` : Microsoft SQL 데이터베이스의 데이터 정렬 방식 - ```azureMysqlCol```


- ```firewall_rule``` 
  + ```start_ip_address``` : 방화벽 Rule에 연결된 시작 IP 주소
  + ```end_ip_address``` : 방화벽 Rule에 연결된 종료 IP 주소
