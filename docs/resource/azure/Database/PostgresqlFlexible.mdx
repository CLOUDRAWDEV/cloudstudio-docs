## 리소스 개요
Azure Database for PostgreSQL 유연한 서버는 유연한 서버 아키텍처를 제공하는 Azure의 확장 가능한 완전 관리형 오픈 소스 관계형 데이터베이스 서비스입니다. 
PostgreSQL 환경 구성에 대한 더 많은 제어가 필요한 다양한 워크로드 및 애플리케이션을 지원하도록 설계되었습니다.
유연한 서버를 통해 고객은 특정 요구 사항을 충족하기 위해 다양한 컴퓨팅 및 스토리지 옵션 중에서 선택할 수 있습니다. 
단일 노드 및 고가용성 구성을 모두 지원하고 백업 및 재해 복구를 위한 여러 옵션을 제공합니다.

## 리소스 설정 값
- ```version``` : 사용할 Postgresql 버전 - ```11```, ```12```, ```13```, ```14```

- ```sku_name``` : Postgresql 가격 책정 계층 이름, Tier + 컴퓨팅 세대 + 코어 수의 조합 - ```azurePostgresqlFlexibleSku```

- ```storage_mb``` : Postgresql Flexible Server의 최대 용량(MB) - ```storageGB```

- ```admin_username``` : Creation mode가 default인 경우 Server의 관리자 사용자 이름

- ```admin_password``` : Creation mode가 default인 경우 Server의 관리자 로그인 암호

- ```auto_grow_enabled``` : 스토리지 자동 증가 활성화 - ```true``` , ```false```

- ```availability_zone``` : 가용 영역을 지정합니다. 가능한 값은 1, 2, 3입니다. - ```1``` ~ ```3```

- ```enable_high_availability``` : Postgresql Flexible Server의 고가용성 활성화 여부  - ```true``` , ```false```

- ```high_availability_mode``` : Postgresql Flexible Server의 고가용성 모드  - ```SameZone``` , ```ZoneRedundant```

- ```standby_availability_zone``` : 대기 서버의 가용 영역을 지정합니다. 가능한 값은 1, 2, 3입니다. 가용 영역과 중복될 수 없습니다.  - ```1``` ~ ```3```

- ```backup_retention_days``` : MySQL Flexible Server의 백업 보존 일수입니다. 가능한 값은 1~35일입니다.  - ```1``` ~ ```35```

- ```geo_redundant_backup_enabled```  - ```true``` , ```false```


- ```public_network_access_enabled``` : Server에 퍼블릭 네트워크 접근 허용 - ```true``` , ```false```

- ```delegated_subnet_name``` : 가상 네트워크 위임을 위한 Subnet 이름

- ```private_dns_zone_name``` : Postgresql Flexible Server를 생성하기 위한 프라이빗 DNS 영역

- ```configuration``` 
  + ```name``` : Postgresql 유연한 서버 구성의 이름
  + ```param_value``` : Postgresql 유연한 서버 구성의 값

- ```database```
  + ```charset``` : Postgresql Database의 문자 집합 - ```azurePostgresqlChartSet``` 
  + ```collation``` : Postgresql Database의 데이터 정렬 방식 - ```azurePostgresqlFlexibleCol```


- ```firewall_rule``` 
  + ```start_ip_address``` : 방화벽 Rule에 연결된 시작 IP 주소
  + ```end_ip_address``` : 방화벽 Rule에 연결된 종료 IP 주소
