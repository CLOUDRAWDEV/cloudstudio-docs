## 리소스 개요

Azure SQL Database는 널리 사용되는 Microsoft SQL Server 엔진을 기반으로 하는 Microsoft Azure에서 제공하는 완전히 관리되는 관계형 데이터베이스 서비스입니다.
하드웨어 또는 소프트웨어 인프라를 관리할 필요 없이 클라우드에서 SQL Server 데이터베이스를 만들고 관리할 수 있습니다.
Azure SQL 데이터베이스를 사용하면 자동 패치, 백업, 고가용성 및 재해 복구와 같은 기능을 활용할 수 있습니다.
또한 지능형 성능 모니터링, 자동 조정, 고급 보안 기능과 같은 기능을 제공하여 데이터베이스 성능을 최적화하고 중요한 데이터를 보호할 수 있습니다.

## 리소스 설정 값

-   `version` : 사용할 Microsoft SQL 서버 버전 - `2.0`, `12.0`

-   `admin_username` : 서버의 관리자 사용자 이름

-   `admin_password` : 서버의 관리자 로그인 암호

-   `minimum_tls_version` : 서버에 지원되는 최소 TLS 버전 - `1.0`, `1.1`, `1.2`

-   `public_network_access_enabled` : 서버에 퍼블릭 네트워크 접근 허용 - `true`, `false`

-   `firewall_rule`

    -   `start_ip_address` : 방화벽 Rule에 연결된 시작 IP 주소
    -   `end_ip_address` : 방화벽 Rule에 연결된 종료 IP 주소

-   `security_alert_policy`

    -   `alert_policy_state` : 적용할 정책의 활성화 상태
    -   `disabled_alerts` : 비활성화할 보안 경고 목록 - `Sql_Injection`, `Sql_Injection_Vulnerability`, `Access_Anomaly`, `Data_Exfiltration`, `Unsafe_Action`
    -   `alert_retention_days` : 위협 탐지 감사 로그의 보관일 수 - `0` ~ `3265`
    -   `alert_email_account_admins` : 이메일에 보안 경고 전송 활성화 - `true` , `false`
    -   `alert_email_addresses` : 보안 경고를 받을 이메일 주소 목록

-   `auditing_policy`

    -   `enable_storage_destination` : Storage 계정 활성화 여부 - `true` , `false`
    -   `enable_log_analytics_destination` : Log analytics 활성화 여부 - `true` , `false`

-   `tag` : 리소스를 카테고리로 분류하기 위한 태그 - 최대 512자, `key`:`value`
