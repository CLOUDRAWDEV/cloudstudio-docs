## 리소스 개요

가용성 집합은 중복성과 가용성을 제공하기 위해 Azure에서 애플리케이션이 빌드되는 방식을 이해할 수 있도록 하는 VM의 논리적 그룹입니다.  
고가용성 애플리케이션을 제공하고 99.95% Azure SLA를 충족할 수 있도록 가용성 집합 내에 둘 이상의 VM을 만드는 것이 좋습니다.  
가용성 집합 자체에 대한 비용은 없으므로 만드는 각 VM 인스턴스에 대해서만 요금을 지불합니다.

## 리소스 설정 값

-   `update_domain_count` : 가상머신 그룹을 동시에 다시 부팅하기 위한 업데이트 도메인의 수 - `1` ~ `20`
-   `fault_domain_count` : Cognitive Service 사용을 위한 가격 책정 계층 - `1` ~ `3`
-   `tag` : 리소스를 카테고리로 분류하기 위한 태그 - 최대 512자, `key`:`value`
