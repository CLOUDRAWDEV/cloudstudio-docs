## 리소스 개요
ExpressRoute Circuit은 사용자가 온-프레미스 인프라와 Azure 데이터 센터 간에 비공개 연결을 만들 수 있도록 Microsoft Azure에서 제공하는 서비스입니다. 
트래픽이 공용 인터넷 대신 개인 연결을 통해 라우팅되므로 일반적인 인터넷 기반 연결보다 더 안전하고 안정적인 연결을 제공합니다.
ExpressRoute Circuit은 기본적으로 사용자가 자체 온-프레미스 인프라의 일부인 것처럼 가상 머신, 저장소 및 데이터베이스와 같은 Azure 서비스에 액세스할 수 있는 전용 프라이빗 네트워크 연결입니다. 
이를 통해 조직은 네트워크를 공용 인터넷에 노출하지 않고도 온-프레미스 인프라를 클라우드로 확장하고 Azure 서비스에 액세스할 수 있습니다.

## 리소스 설정 값
- ```service_provider``` : Express Route 공급을 제공할 ISP 공급자 -```KINX```,  ```KT```,  ```SKTTelecom```,  ```LGCNS```,  ```LGUplus````,  ```SejongTelecom```

- ```peering_location``` : ISP가 연결을 프로비전할 Peering 위치, Azure resource location과 다른 의미  - ```Seoul```,  ```Busan```

- ```bandwidth_in_mbps``` : 데이터 전송 대역폭(Mbps) -```50```,  ```100```,  ```200```,  ```500```,  ```1000````,  ```2000```,  ```5000````,  ```10000```

- ```sku_tier``` : Express Route 회로의 가격 책정 계층 - ```Basic```,  ```Local```,  ```Standard```,  ```Premium```

- ```sku_family``` : 아웃바운드 데이터 전송을 위한 청구 모델 - ```MeteredData```,  ```UnlimitedData```

- ```authorization_name``` : 회로를 사용할 수 있는 인증 이름 

- ```circuit_peering``` 
    + ```peering_type``` : Express Route 회로 피어링 유형 - ```AzurePrivatePeering```,  ```MicrosoftPeering```
    + ```peer_asn``` : 16비트의 ASN, 공용 혹은 프라이빗 - ```0```~```65535```
    + ```primary_peer_address``` : 주 링크에 대한 /30 IPv4 서브넷 
    + ```secondary_peer_address``` : 보조 링크에 대한 /30 IPv4 서브넷
    + ```vlan_id``` : 피어링 설정을 위한 유효한 VLAN ID
    + ```advertised_public_prefixes``` : BGP 세션을 통해 보급된 모든 공용 IPv4 주소 접두사의 목록
    + ```included_circuit_name``` : 피어링을 설정할 Express Route 회로 이름
