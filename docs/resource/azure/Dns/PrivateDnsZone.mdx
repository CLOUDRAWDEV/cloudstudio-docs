## 리소스 개요
Azure 프라이빗 DNS는 사용자 지정 DNS 솔루션을 추가하지 않고도 가상 네트워크의 도메인 이름을 관리하고 확인할 수 있는 안정적이고 신뢰할 수 있는 DNS 서비스를 제공합니다. 
프라이빗 DNS 영역을 사용하면 현재 Azure에서 제공하는 이름 대신 사용자 고유의 사용자 지정 도메인 이름을 사용할 수 있습니다.
프라이빗 DNS 영역에 포함된 레코드는 인터넷에서 확인할 수 없습니다. 프라이빗 DNS 영역에 대한 DNS 확인은 연결된 가상 네트워크에서만 작동합니다.

## 리소스 설정 값
- ```private_dns_a_record```
  + ```ttl``` : DNS Record의 Time To Live(초) - ```0``` ~ ```2147483647```
  + ```records``` : IPv4 주소의 목록


- ```private_dns_aaaa_record```
  + ```ttl``` : DNS Record의 Time To Live(초) - ```0``` ~ ```2147483647```
  + ```records``` : IPv6 주소의 목록


- ```private_dns_cname_record```
  + ```ttl``` : DNS Record의 Time To Live(초) - ```0``` ~ ```2147483647```
  + ```record``` : 하위 도메인을 다른 레코드에 연결하기 위한 별칭


- ```private_dns_ptr_record```
  + ```ttl``` : DNS Record의 Time To Live(초) - ```0``` ~ ```2147483647```
  + ```records``` : 포인터 레코드 유형의 도메인 이름 목록


- ```private_dns_txt_record```
  + ```ttl``` : DNS Record의 Time To Live(초) - ```0``` ~ ```2147483647```
  + ```records``` : 텍스트 유형의 레코드 목록


- ```vnet_link```
  + ```linked_vnet_name``` : DNSRecord의 Time To Live(초)
